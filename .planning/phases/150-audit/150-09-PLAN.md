       - Render pagination only for the active tab.

    - Confirm `getDepartmentRisks/Controls/KRIs` accept `skip` and `limit` and forward them.
    - Do not change response shapes (still list).

    In `frontend/src/services/userApi.ts`:
    - Add `department_id?: number` to `listVisibleUsers` params and pass it through to `/users/lookup`.
    - Keep current response type (array of user lookup objects).

    Logic-flow safety checks:
    - Reset `riskPage` to 1 when `riskFilter` changes so pagination doesn’t request a page that no longer exists.
    - Fetch department metadata (`departmentApi.getDepartment`) once on `id` change; fetch list data when the active tab page changes to avoid refetching all tabs.

    Example frontend structure (adapt to your component layout):

      if (!id) return;
      departmentApi.getDepartment(Number(id)).then(setDepartment);
    }, [id]);

    useEffect(() => {
      if (!id || activeTab !== \"risks\") return;
      const skip = (riskPage - 1) * DEPARTMENT_PAGE_SIZE;
      departmentApi.getDepartmentRisks(Number(id), { skip, limit: DEPARTMENT_PAGE_SIZE, min_net_score })
        .then(setRisks);
    }, [id, activeTab, riskPage, min_net_score]);
