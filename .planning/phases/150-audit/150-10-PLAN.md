  <name>Task 1: Apply department_id filter for manager lookup results</name>
  <files>backend/app/api/v1/endpoints/users.py, backend/tests/test_users.py</files>
  <action>
    In `/users/lookup`, update the AccessScope.MANAGER branch so a provided `department_id` actually narrows the returned users.

    Current behavior only blocks mismatched department_id, but still returns self + direct reports without filtering by department. This breaks Department Detail, which expects users only from the selected department.

    Implementation details:
    1) Keep the existing guard that returns `[]` when `department_id` is provided and does not match `current_user.department_id`.
    2) After building the manager scope query, add a department filter when `department_id` is provided.
    3) Preserve default behavior when `department_id` is not provided (self + direct reports across departments).

    Example snippet (shape only, keep existing imports/logic):
    ```python
    elif current_user.access_scope == AccessScope.MANAGER:
        if department_id is not None and department_id != current_user.department_id:
            return []
        query = query.where(or_(User.id == current_user.id, User.manager_id == current_user.id))
        if department_id is not None:
            query = query.where(User.department_id == department_id)
    ```

    Add regression coverage in `backend/tests/test_users.py`:
    - Create dept A + dept B.
    - Create a manager in dept A with AccessScope.MANAGER.
    - Create one direct report in dept A and one in dept B (manager_id = manager.id).
    - Call `/api/v1/users/lookup?department_id={dept_a.id}` with X-Mock-User-Id = manager.
    - Assert results include only users from dept A (manager + in-dept report), exclude the dept B report.

    Example test outline:
    ```python
    response = await client.get(
        f"/api/v1/users/lookup?department_id={dept_a.id}",
        headers={"X-Mock-User-Id": str(manager.id)},
    )
    ids = {u["id"] for u in response.json()}
    assert report_a.id in ids
    assert report_b.id not in ids
    ```
  </action>
  <verify>`pytest backend/tests/test_users.py`</verify>
  <done>Manager lookup honors department_id filtering and never returns cross-department reports when a department filter is provided.</done>
