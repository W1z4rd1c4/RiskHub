<task type="auto">
  <name>Task 2: Standardize pagination limits + deterministic user lookup paging</name>
  <files>backend/app/core/pagination.py, backend/app/api/v1/endpoints/users.py, backend/app/api/v1/endpoints/departments.py, backend/tests/test_users.py</files>
  <action>
    Create `backend/app/core/pagination.py`:
    - `DEFAULT_PAGE_SIZE = 50`
    - `MAX_PAGE_SIZE = 100`
    - `MAX_LOOKUP_SIZE = 200`

    Update `/users/lookup` in `backend/app/api/v1/endpoints/users.py`:
    1) Replace hardcoded `Query` limits with `Query(DEFAULT_PAGE_SIZE, le=MAX_LOOKUP_SIZE)` for `limit`.
    2) Add `order_by(User.id)` before `.offset(skip).limit(limit)` to ensure stable paging.
    3) Add optional `department_id: int | None = None` query param.
       - If provided, apply `User.department_id == department_id` ONLY if the caller is allowed to see that department:
         - GLOBAL: allow any department_id.
         - DEPARTMENT: allow only if department_id == current_user.department_id.
         - MANAGER: allow only if department_id == current_user.department_id (or ignore if none).
       - If not allowed, return an empty list (avoid leaking existence via 403).

    Update department detail list endpoints in `backend/app/api/v1/endpoints/departments.py`:
    - `list_department_risks`, `list_department_controls`, `list_department_kris` should use
      `Query(DEFAULT_PAGE_SIZE, le=MAX_PAGE_SIZE)` for `limit`.
    - Do NOT change `list_departments` (it is not paginated).

    Example lookup paging snippet:
    ```python
    limit: int = Query(DEFAULT_PAGE_SIZE, ge=1, le=MAX_LOOKUP_SIZE)
    ...
    if department_id is not None:
        if current_user.access_scope == AccessScope.GLOBAL:
            query = query.where(User.department_id == department_id)
        elif current_user.department_id == department_id:
            query = query.where(User.department_id == department_id)
        else:
            return []
    result = await db.execute(query.order_by(User.id).offset(skip).limit(limit))
    ```

    Add tests in `backend/tests/test_users.py`:
    - Paging determinism: create 3+ users, request `/users/lookup?limit=2&skip=0` then `/users/lookup?limit=2&skip=2` and assert no overlap.
    - Department scoping: employee user from dept A requesting `department_id` of dept B returns empty list.
    - Limit enforcement: request `limit=9999` and assert response length <= MAX_LOOKUP_SIZE.

    Test snippet example:
    ```python
    resp1 = await client_employee.get(\"/api/v1/users/lookup?limit=2&skip=0\")
    resp2 = await client_employee.get(\"/api/v1/users/lookup?limit=2&skip=2\")
    ids1 = {u[\"id\"] for u in resp1.json()}
    ids2 = {u[\"id\"] for u in resp2.json()}
    assert ids1.isdisjoint(ids2)
    ```
  </action>
