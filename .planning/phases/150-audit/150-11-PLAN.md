    Add an explicit `useEffect` that resets pagination state whenever `id` changes. This ensures all tabs (risks/controls/kris/users) return to page 1 when navigating between departments.

    Implementation details:
    - Keep the existing risk filter reset, but remove `id` from its dependency list to avoid double resets.
    - Add a new effect that runs on `[id]` and resets `riskPage`, `controlPage`, `kriPage`, and `userPage` to 1.
    - Do not change `activeTab` or `riskFilter`; only reset the page counters.

    Example snippet:
    ```tsx
    useEffect(() => {
      if (!id) return;
      setRiskPage(1);
      setControlPage(1);
      setKriPage(1);
      setUserPage(1);
    }, [id]);

    useEffect(() => {
      setRiskPage(1);
    }, [riskFilter]);
    ```

    This guarantees that switching departments never leaves the UI on an out-of-range page.
