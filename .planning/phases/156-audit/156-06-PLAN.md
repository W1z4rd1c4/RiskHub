<task type="auto">
  <name>Task 1: Make CSP environment-aware (remove unsafe-eval in production)</name>
  <files>backend/app/middleware/security.py</files>
  <action>
Adjust `SecurityHeadersMiddleware` so CSP is strict in production and permissive only in debug:

- In `settings.debug == True`:
  - Keep the current dev-friendly directives (including `'unsafe-eval'` if required for dev tooling).
- In `settings.debug == False`:
  - Remove `'unsafe-eval'` from `script-src`.
  - Narrow `connect-src` away from `https://*` where possible.
    - Minimum safe approach: `connect-src 'self'` (since backend serves API and should be same-origin with frontend in typical deployments).
    - If your deployment needs additional origins, derive them from `settings.cors_origins` (strip scheme/paths carefully and emit CSP-safe sources).

Avoid:
- Tightening in debug in a way that breaks local dev.
- Shipping `'unsafe-eval'` in production.
  </action>
  <verify>cd backend && pytest -q</verify>
  <done>Production responses include CSP without `'unsafe-eval'`; debug mode remains functional.</done>
</task>

<task type="auto">
  <name>Task 2: Honor LOG_LEVEL for file handlers (avoid forced DEBUG)</name>
  <files>backend/app/core/logging.py</files>
  <action>
Update logging configuration so file handlers and root logger respect `LOG_LEVEL`:

- Today, `app_handler`, `audit_handler`, and `root_logger` are forced to DEBUG.
- Change behavior:
  - Parse `log_level` once (already passed into `configure_logging`).
  - Set:
    - `root_logger.setLevel(parsed_level)`
    - `app_handler.setLevel(parsed_level)` (or a slightly more verbose level if explicitly desired)
    - `audit_handler.setLevel(parsed_level)` (or INFO minimum, depending on compliance requirements)

Constraints:
- Do not break existing admin log endpoints that read and parse logs.
- Keep JSON formatting unchanged.

Avoid:
- Making logs silent; INFO should remain visible.
- Inconsistent levels between console and file unless explicitly justified.
  </action>
  <verify>cd backend && pytest -q</verify>
  <done>When `LOG_LEVEL=INFO`, logs written to files are not DEBUG-level spam.</done>
</task>
