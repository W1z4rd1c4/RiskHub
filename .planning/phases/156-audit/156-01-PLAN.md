<task type="auto">
  <name>Task 1: Enforce RBAC before applying `department_id` filter</name>
  <files>backend/app/api/v1/endpoints/kris.py</files>
  <action>
Update `/kris/overdue` and `/kris/due-soon` so that an explicit `department_id` filter cannot bypass department scoping:

- Compute `dept_ids = get_user_department_ids(current_user)` before any early-return filtering.
- If `department_id` is provided:
  - If `dept_ids is None` (privileged/global): allow it.
  - Else if `department_id` is in `dept_ids`: allow it.
  - Else: return `[]` (do NOT raise 403) to avoid leaking department existence and to match existing “safe filter” patterns used elsewhere.
- Only after the above check, filter the computed list by `department_id`.

Avoid:
- Filtering by `department_id` before RBAC checks.
- Returning data for unauthorized department IDs.
- Returning different error shapes for unauthorized filter values (stay consistent: return `[]`).
  </action>
  <verify>cd backend && pytest -q tests/test_kris_history_api.py -k "overdue or due_soon"</verify>
  <done>`/kris/overdue?department_id=X` and `/kris/due-soon?department_id=X` never return items outside the caller’s scope.</done>
</task>
