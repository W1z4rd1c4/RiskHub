Implement a strict production guard:

1) In `directory.py` signature verification:
   - If `settings.webhook_secret` is empty:
     - If `settings.debug` is True: keep current “warn and skip” behavior (dev convenience).

- Breaking dev workflows (DEBUG=true must remain usable without a secret).
- Returning 200/“processed” when secret is unset in production mode.

<task type="checkpoint:decision" gate="blocking">
  <decision>Decide whether production should fail-fast on missing WEBHOOK_SECRET at startup</decision>
  <context>
Rejecting the webhook endpoint in production is the minimum safe behavior and does not break deployments that don’t use directory sync.
Failing app startup is stricter, but could block installs where directory sync is not configured/used.
  </context>
  <options>
    <option id="no-startup-failfast">
      <name>No startup fail-fast (default)</name>
      <pros>Does not block deployments that don’t use directory webhooks; still safe because endpoint rejects in production without secret</pros>
      <cons>Misconfiguration discovered only when webhook is invoked</cons>
    </option>
    <option id="startup-failfast-flagged">
      <name>Startup fail-fast behind an explicit env flag</name>
      <pros>Security testing can enable strict behavior; production can opt in when directory sync is mandatory</pros>
      <cons>Adds one more config toggle to document</cons>
    </option>
  </options>
  <resume-signal>Select: no-startup-failfast or startup-failfast-flagged</resume-signal>
</task>
