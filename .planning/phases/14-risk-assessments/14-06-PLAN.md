  <name>Task 1: Add CRO-only backend endpoint for batch send</name>
  <files>backend/app/api/v1/endpoints/riskhub_questionnaires.py, backend/app/api/v1/router.py</files>
  <action>
Create a CRO-only endpoint module under the Risk Hub API namespace (to reuse `require_cro` semantics):
- Include router as `/riskhub/questionnaires` (new include in `router.py`).

Add `POST /riskhub/questionnaires/batch-send` accepting:
- `select_all: bool`
- `risk_ids?: int[]` (required when `select_all=false`)
- `filters?: { department_id?, process?, category?, status? }` (required when `select_all=true`)

Behavior:
- Resolve the target risk list:
  - If `select_all=true`, query risks using the same filter semantics as Risks list (department/process/category + exclude archived).
  - Else, use explicit `risk_ids`.
- For each risk:
  - Skip risks with `owner_id is null` and record in `skipped_no_owner`.
  - Skip risks that already have an open questionnaire and record in `skipped_open_exists`.
  - Otherwise create a new questionnaire instance (same helper used by single send).
- Return a structured summary:
  - `created_count`, `skipped_no_owner`, `skipped_open_exists`, `errors[]`.

This endpoint is CRO-only; RM bulk send is not required by the product spec.
  </action>
  <verify>cd backend && python -m compileall app</verify>
  <done>Backend supports batch sending with select-all + filters and returns a deterministic summary.</done>
