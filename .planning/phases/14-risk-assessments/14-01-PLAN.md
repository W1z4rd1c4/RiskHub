  <action>
Create a new model `RiskQuestionnaire` representing a single тАЬsent questionnaireтАЭ instance linked to a Risk.

Recommended columns:
- `id` (PK)
- `risk_id` (FK тЖТ `risks.id`, indexed)
- `assigned_to_user_id` (FK тЖТ `users.id`, indexed) тАФ defaults to Risk.owner_id at send time
- `sent_by_user_id` (FK тЖТ `users.id`) тАФ RM/CRO who initiated
- `status` enum: `sent`, `in_progress`, `submitted`
- `template_key` (string, e.g. `risk_owner_reassessment`)
- `template_version` (string, e.g. `v1`)
- `answers` JSON (nullable until draft/submission)
- timestamps:
  - `sent_at` (UTC, tz-aware)
  - `due_at` (UTC, tz-aware)
  - `submitted_at` (nullable)
  - `submitted_by_user_id` (nullable FK тЖТ users)
  - `created_at`, `updated_at` (optional; follow existing model conventions)

Indexes:
- `(risk_id, status)` to query тАЬopen questionnaire for riskтАЭ

- `(due_at, status)` to power reminder jobs

Add the model to `backend/app/models/__init__.py` exports so it is importable via `from app.models import RiskQuestionnaire`.

- Create the `risk_questionnaires` table + indexes.
- Keep enum constraints consistent with existing patterns (prefer `SAEnum(..., native_enum=False, validate_strings=True)` or `SQLEnum(..., create_constraint=True)` as used elsewhere).
