<task type="auto">
  <name>Task 1: Add backend endpoints for risk and breach trend series</name>
  <files>backend/app/schemas/dashboard.py, backend/app/api/v1/endpoints/dashboard.py, backend/tests/api/v1/test_dashboard_history.py</files>
  <action>
    Add schemas in backend/app/schemas/dashboard.py:
    - class RiskTrendPoint { period: str; total_new: int; critical_new: int }
    - class KRIBreachTrendPoint { period: str; total_entries: int; breached_entries: int }
    
    Implement GET /dashboard/risk-trends:
    - Query params: department_id?, include_archived? (same behavior as summary).
    - Build RBAC conditions using get_user_department_ids.
    - Use date_trunc('month', Risk.created_at) and to_char for period label (YYYY-MM).
    - Select total_new = count(Risk.id) and critical_new = sum(case(Risk.net_score >= 15, 1, else 0)).
    - Group by period, order asc, limit to last 12 months (use order_by then slice).
    - If dept_ids is empty list, return [] immediately.
    
    Implement GET /dashboard/kri-breach-trends:
    - Join KRIValueHistory -> KeyRiskIndicator -> Risk.
    - Filter Risk.status != archived, apply dept scoping, and ignore entries with null period_end.
    - Group by date_trunc('month', KRIValueHistory.period_end), label as YYYY-MM.
    - total_entries = count(KRIValueHistory.id), breached_entries = sum(case(breach_status != 'within', 1, else 0)).
    - Order asc and limit to last 12 months.
    
    Optional: fill missing months with zero counts so charts do not skip months.
    
    Tests (backend/tests/api/v1/test_dashboard_history.py):
    - Create risks with created_at in two explicit months and verify total_new and critical_new.
    - Create KRIValueHistory entries across two months with mixed breach_status.
    - Assert sorted period order and that no-access users (empty dept_ids) get [].
  </action>
  <verify>cd backend && pytest tests/api/v1/test_dashboard_history.py -v</verify>
  <done>New dashboard endpoints return ordered trend series with correct scoping</done>
</task>
