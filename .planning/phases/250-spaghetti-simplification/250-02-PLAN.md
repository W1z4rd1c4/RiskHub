# Plan 250-02: Simplify `dashboard.py` (quarterly comparison)

## Objective
Refactor the `GET /quarterly-comparison` implementation in `backend/app/api/v1/endpoints/dashboard.py` (1144 lines) to be readable and testable by:
- extracting the nested helpers and heavy logic into a small service/module, and
- removing obviously dead/unreturned computations inside `get_quarter_metrics`.

Primary goal: **preserve exact metric semantics** (time boundaries, scoping, output keys).

## Non-goals / Invariants (do not change)
- Do not change returned JSON structure (`this_quarter`, `last_quarter`, `changes`, `period`, `snapshot_info`).
- Do not change quarter boundary semantics:
  - Period metrics use **end-exclusive** intervals: `[start, end)`.
  - For the in-progress current quarter, `end` is **now**.
  - Datetimes used for DB queries remain **naive UTC** (`now_utc.replace(tzinfo=None)`).
- Do not introduce generic archived filters (risk uses `status != archived`; KRI uses `is_archived == False`).
- Do not change department scoping behavior (`get_user_department_ids(current_user)` semantics).

## Current State (verified)
- **File**: `backend/app/api/v1/endpoints/dashboard.py`
- `get_quarterly_comparison` contains nested helpers:
  - `parse_quarter()`
  - `get_quarter_metrics()`
  - `get_snapshot_metrics()` (delegates to `app.core.snapshot_service.capture_snapshot_metrics`)
  - `get_historical_snapshot()` (delegates to `app.core.snapshot_service.get_quarter_snapshot`)
- `get_quarter_metrics()` currently computes several values it **does not return** (dead work); only these keys are returned today:
  - `new_risks`, `closed_risks`, `audit_activity`, `failed_audits`, `unaudited_controls`, `activity_volume`

## Implementation Steps

### 1) Extract quarterly comparison logic into a small service module
- [CREATE] `backend/app/services/quarterly_comparison_service.py` (preferred)  
  - Implement a single entrypoint:
    - `async def build_quarterly_comparison(db: AsyncSession, current_user: User, current_quarter: str | None, compare_quarter: str | None) -> dict`
  - Move these helpers out of the endpoint (keep behavior identical):
    - `parse_quarter()`
    - quarter boundary calculation for `current_quarter_start/end` and `last_quarter_start/end`
    - `get_quarter_metrics()` (period-based metrics only)
    - the “snapshot fallback” behavior and `snapshot_metrics` handling
  - Keep calls to `app.core.snapshot_service` as-is (do not duplicate snapshot logic in this new module).

- [MODIFY] `backend/app/api/v1/endpoints/dashboard.py`
  - Replace the body of `get_quarterly_comparison()` with a thin orchestration call to `build_quarterly_comparison(...)`.

### 2) Remove dead computations from `get_quarter_metrics()` (safe simplification)
- In the extracted `get_quarter_metrics()` helper:
  - Remove any computed variables not used in the returned dict.
  - Keep **only** the period-based metrics the endpoint returns today.
  - Do not “upgrade” metrics from period-based to snapshot-based or vice-versa.

### 3) Keep imports clean and deterministic
- Eliminate repeated inner imports where safe (move to module top in the service module).
- Keep any imports inside functions only if they avoid circular dependencies.
