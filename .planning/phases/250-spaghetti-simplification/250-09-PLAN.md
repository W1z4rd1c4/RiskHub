# Plan 250-09: Simplify `DepartmentDetailPage.tsx` (data hook + tab panels)

## Objective
Refactor `frontend/src/pages/DepartmentDetailPage.tsx` (~574 lines) to reduce state/effect sprawl by:
- extracting the data-fetching logic into a dedicated hook, and
- extracting tab panels into small components (or at least local render functions) while preserving behavior.

## Non-goals / Invariants (do not change)
- Do not change API calls or query param semantics:
  - `departmentApi.getDepartment(id)`
  - `departmentApi.getDepartmentRisks(id, { skip, limit, min_net_score? })`
  - `departmentApi.getDepartmentControls(id, { skip, limit })`
  - `departmentApi.getDepartmentKRIs(id, { skip, limit })`
  - `userApi.listVisibleUsers({ department_id, skip, limit })`
- Keep pagination semantics:
  - page size = `DEPARTMENT_PAGE_SIZE = 100`
  - skip = `(page - 1) * DEPARTMENT_PAGE_SIZE`
- Keep “high risk” filter semantics:
  - when `riskFilter === 'high'`, pass `min_net_score = 10` (current constant)
- Keep row click navigation targets unchanged.

## Current State (verified)
- Department metadata is fetched once on `id` change.
- Each tab has its own `useEffect` to fetch data when active and when its page changes.
- Per-tab pagination state exists (`riskPage`, `controlPage`, `kriPage`, `userPage`).
- `activity` tab uses `department.recent_executions` (no separate fetch).

## Implementation Steps

### 1) Extract data fetching into a hook
- [CREATE] `frontend/src/hooks/useDepartmentDetail.ts`
  - Inputs:
    - `departmentId: number | undefined`
    - `activeTab: 'risks' | 'controls' | 'kris' | 'activity' | 'users'`
    - current pages + `riskFilter`
  - Outputs:
    - `department`, `isLoadingDepartment`, `error`
    - tab data arrays: `risks`, `controls`, `kris`, `users`
    - derived pagination totals (`riskTotalPages`, `controlTotalPages`, `kriTotalPages`, `userTotalPages`)
    - page setters may remain in the page; the hook just reads them
  - Internals:
    - Keep the same “fetch only when tab is active” behavior.
    - Keep the “reset risk page to 1 when riskFilter changes” behavior (either in hook or page).

### 2) Extract tab panels (keep lean)
Pick one of these approaches (prefer the leaner one unless the JSX is still hard to read):
- **Option A (leanest):** extract local render functions inside the page file:
  - `renderRisksTab()`, `renderControlsTab()`, `renderKrisTab()`, `renderUsersTab()`, `renderActivityTab()`
- **Option B:** extract components:
  - [CREATE] `frontend/src/components/departments/DepartmentRisksTab.tsx` (and similarly for others)
  - Keep them presentational: receive `data`, `columns`, `pagination` props and render `SortableTable` + `Pagination`.

### 3) Refactor page to orchestrate
- [MODIFY] `frontend/src/pages/DepartmentDetailPage.tsx`
  - Keep:
    - tab buttons and `activeTab` state
    - columns definitions (or move to a `columns.ts` only if it materially improves readability)
  - Replace the multiple fetch `useEffect`s with the hook.

Manual smoke:

- Switching tabs triggers loading of that tab’s data and pagination works.
- Risk filter “high” reduces the list and the count used for pagination.
- Activity tab shows recent executions and navigation works.

- If anything regresses, revert and re-introduce the hook for only one tab first (risks), then expand.

- Fewer `useEffect` blocks in `DepartmentDetailPage.tsx`.
- Clear separation: data loading in hook, rendering in page/panels.
- No behavior changes; build passes.
