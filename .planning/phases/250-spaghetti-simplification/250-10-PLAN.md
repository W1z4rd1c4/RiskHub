# Plan 250-10: Service layer simplification (Directory Sync + Orphaned Items)

## Objective
Simplify and de-spaghetti the service layer code in:
- `backend/app/services/directory_sync_service.py` (~674 lines)
- `backend/app/services/orphaned_item_service.py` (~663 lines)

…with a bias toward **lean code**: prefer small, explicit helper functions over inheritance/base classes.

## Non-goals / Invariants (do not change)
- Do not change transaction boundaries in ways that change behavior:
  - `DirectorySyncService.sync_single_user(... user.deactivated ...)` currently:
    - flags orphaned items (flushes) and commits when user is deactivated
    - that commit must still persist both the user status and created orphan records
- Do not change directory sync semantics:
  - preview vs apply behavior
  - sync log status updates on failure
- Do not change orphaned-item resolution semantics (who/what gets reassigned, fallback department logic).
- Avoid introducing a `BaseService` class unless it demonstrably removes *real* duplication across ≥2 services.

## Current State (verified highlights)
- `DirectorySyncService` has a large `_run_sync(...)` and a `sync_single_user(...)` with branching per event type.
- `sync_single_user(...)` has some obvious low-risk cleanup opportunities (e.g., duplicated normalization assignment).
- `OrphanedItemService` contains several long “scan/flag” loops with repeated “check existing → create orphan record” patterns.

## Implementation Steps

### 0) First: write down *concrete* duplicated patterns you will remove
Before creating any new shared module/class:
- List the exact duplicated blocks you intend to de-duplicate and where they occur.
- If the only commonality is “uses AsyncSession + logger”, skip shared abstractions.

### 1) Simplify `directory_sync_service.py` by extracting local helpers (no base class)
- [MODIFY] `backend/app/services/directory_sync_service.py`
  - Extract private helpers for clarity (examples):
    - `_load_local_state(db)` → returns users/departments/caches
    - `_mark_sync_failed(sync_log, error_msg)` (ensures consistent sync_log updates)
  - Remove low-risk duplication:
    - eliminate repeated assignments (e.g., duplicated `target_department = _normalize_text(...)`)
  - Keep public API unchanged: `preview_sync`, `apply_sync`, `sync_single_user`

### 2) Simplify `orphaned_item_service.py` by extracting internal helpers
- [MODIFY] `backend/app/services/orphaned_item_service.py`
  - Extract private helpers for repeated patterns, e.g.:
    - `_already_flagged(db, item_type, item_id) -> bool`
    - `_create_orphan(db, item_type, item_id, previous_owner_id, *, orphaned_at=...)`
  - Use helpers to reduce repeated “existing check + create record” boilerplate in:
    - `flag_orphaned_items`
    - `scan_uncategorised_items`
  - Keep method signatures and return values unchanged.

### 3) Only if truly shared: add a tiny shared helper module (function-based)
If (and only if) you find a helper that is used by **both** services (and is not trivial), create:
- `backend/app/services/_service_utils.py` with 1–2 functions max
- Avoid `BaseService` inheritance; prefer `def` helpers.
