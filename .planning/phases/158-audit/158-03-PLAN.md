<task type="auto">
  <name>Task 3: Add a forward-safe migration to (re)create ux_approval_pending</name>
  <files>backend/alembic/versions/*_restore_ux_approval_pending.py</files>
  <action>
Create a new Alembic revision (do not edit historical revisions) that ensures the unique partial index exists after head:
- Create `ux_approval_pending` as a UNIQUE index on `(resource_type, resource_id, action_type)`.
- Predicate must cover the real “pending queue” statuses, including historical lowercase drift:
  - `status IN ('PENDING','PENDING_PRIVILEGED','pending_privileged')`

If using `op.create_index`, use `postgresql_where=...`. If using `op.execute`, ensure it’s idempotent (`IF NOT EXISTS`).

Avoid:
- A predicate that matches only one spelling (case drift already happened in migration history).
- Dropping/recreating enum types.

Evidence of the bug in current codebase (wrong migration behavior):

1) `ux_approval_pending` is dropped in an upgrade migration and not recreated (`backend/alembic/versions/6df2bb0adaa3_add_user_preferences_columns.py` around `#L24`):
```py
op.drop_index(
    op.f('ux_approval_pending'),
    table_name='approval_requests',
    postgresql_where="((status = 'PENDING'::approval_status) OR (status = 'PENDING_PRIVILEGED'::approval_status))",
)
```

2) The original partial index migration uses lowercase in predicate (`backend/alembic/versions/h2i3j4k5l6m7_add_partial_unique_index_approval_pending.py` around `#L28`):
```sql
WHERE status = 'PENDING' OR status = 'pending_privileged'
```
But the model enum uses uppercase `PENDING_PRIVILEGED`, so drift exists.

Example fix migration (Postgres) using `op.execute`:
```py
op.execute(
    \"\"\"
    CREATE UNIQUE INDEX IF NOT EXISTS ux_approval_pending
    ON approval_requests (resource_type, resource_id, action_type)
    WHERE status IN ('PENDING','PENDING_PRIVILEGED','pending_privileged')
    \"\"\"
)
```

Example fix migration using `op.create_index`:
```py
op.create_index(
    "ux_approval_pending",
    "approval_requests",
    ["resource_type", "resource_id", "action_type"],
    unique=True,
    postgresql_where="status IN ('PENDING','PENDING_PRIVILEGED','pending_privileged')",
)
```
