Replace lexicographic-only ordering with a length-aware approach:
- Filter by prefix pattern as today.
- Order by `length(risk_id_code)` DESC, then `risk_id_code` DESC.
- Fetch a small set (e.g., 20) and compute max numeric suffix in Python.

This keeps DB work simple while guaranteeing that R100+ beats R99.

Avoid:
- Ordering only by string value.
- Depending on `.limit(100)` being “enough” (it’s not the core problem).

Step-by-step implementation guidance:
1. Add `from sqlalchemy import func` if not already imported in `risks.py`.
2. Update the query to order by `func.length(Risk.risk_id_code).desc()` first.
3. Keep `Risk.risk_id_code.desc()` as a tie-breaker for same-length codes.
4. Keep suffix parsing in Python (as it exists today), but now the “top” returned row(s) are guaranteed to include the numerically largest suffix.

Fix example sketch:
```py
from sqlalchemy import func

result = await db.execute(
    select(Risk.risk_id_code)
    .where(Risk.risk_id_code.like(pattern))
    .order_by(func.length(Risk.risk_id_code).desc(), Risk.risk_id_code.desc())
    .limit(20)
)
```
