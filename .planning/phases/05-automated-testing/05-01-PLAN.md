# Plan 05-01: Backend API Testing with pytest

## Objective
Establish comprehensive test coverage for the FastAPI backend using pytest and pytest-asyncio, covering all critical API endpoints.

## Scope

### In Scope
- pytest + pytest-asyncio setup
- SQLite in-memory test database
- Test coverage for Controls, Risks, Executions, Dashboard endpoints
- Test fixtures for common entities (users, departments, controls, risks)
- CI-ready test commands

### Out of Scope
- Frontend testing (separate plan)
- Performance/load testing
- Authentication testing (mocked auth)

## Tasks

### Task 1: Install Testing Dependencies
Add to `requirements.txt`:
```
pytest>=8.0.0
pytest-asyncio>=0.23.0
httpx>=0.27.0
```

### Task 2: Create Test Configuration
Create `backend/tests/conftest.py`:
- Async test database (SQLite in-memory)
- Database session fixture
- Test client fixture (httpx AsyncClient)
- Entity factory fixtures (user, department, control, risk)

### Task 3: Create Control API Tests
Create `backend/tests/test_controls.py`:
- [ ] `test_create_control` - Create new control
- [ ] `test_get_control` - Retrieve single control
- [ ] `test_list_controls` - List with pagination
- [ ] `test_update_control` - Update control fields
- [ ] `test_delete_control` - Soft delete (archive)
- [ ] `test_link_risk` - Link control to risk
- [ ] `test_unlink_risk` - Remove control-risk link

### Task 4: Create Risk API Tests
Create `backend/tests/test_risks.py`:
- [ ] `test_create_risk` - Create new risk
- [ ] `test_get_risk` - Retrieve single risk
- [ ] `test_list_risks` - List with filters (status, type, priority)
- [ ] `test_update_risk` - Update risk fields
- [ ] `test_delete_risk` - Soft delete (archive)
- [ ] `test_link_control` - Link risk to control

### Task 5: Create Execution API Tests
Create `backend/tests/test_executions.py`:
- [ ] `test_create_execution` - Log new execution
- [ ] `test_list_executions` - List with filters
- [ ] `test_get_execution` - Retrieve single execution

### Task 6: Create Dashboard API Tests
Create `backend/tests/test_dashboard.py`:
- [ ] `test_risk_overview` - /dashboard/risk-overview
- [ ] `test_control_summary` - /dashboard/control-summary
- [ ] `test_risk_matrix` - /dashboard/risk-matrix
- [ ] `test_department_controls` - /dashboard/departments

### Task 7: Add pytest Configuration
Create `backend/pytest.ini`:
```ini
[pytest]
asyncio_mode = auto
testpaths = tests
python_files = test_*.py
```

## Verification

```bash
cd backend
source venv/bin/activate
pytest -v --tb=short
```

Expected: All tests pass with 0 failures.

## Success Criteria
- [ ] All test files created in `backend/tests/`
- [ ] `conftest.py` provides reusable fixtures
- [ ] All endpoint categories have test coverage
- [ ] `pytest -v` runs successfully
- [ ] Tests use isolated in-memory database
