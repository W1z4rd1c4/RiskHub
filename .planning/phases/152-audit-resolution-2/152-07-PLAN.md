# Plan 152-07: Fix Cross-Department Access Inconsistencies

## Priority: 🟡 MEDIUM (functionality gap)

## Problem
Cross-department access via ownership is inconsistent across endpoints.

**Inconsistency identified:**

| Endpoint | KRI Reporting Owner | Control Owner |
|----------|---------------------|---------------|
| `GET /risks/{id}` | ✅ Allowed (line 266) | ✅ Allowed (line 270) |
| `GET /risks/{id}/controls` | ❌ Blocked (line 727) | ❌ Blocked (line 727) |

**Current code in `risks.py`:**
```python
# GET /risks/{id} - has special access checks
if await is_risk_kri_reporting_owner(db, current_user.id, risk_id):
    return risk
if await is_risk_control_owner(db, current_user.id, risk_id):
    return risk

# GET /risks/{id}/controls - only dept check
check_department_access(risk.department_id, current_user)  # ❌ No owner checks
```

**Impact:** User can view a risk via ownership but cannot see its linked controls, breaking expected UX.

---

## Proposed Changes

### [MODIFY] [risks.py](../../../backend/app/api/v1/endpoints/risks.py)

**Lines ~717-727:** Add ownership access checks before department check
```diff
 async def list_risk_controls(
     risk_id: int,
     db: AsyncSession = Depends(get_db),
     current_user: User = Depends(deps.get_current_user),
 ):
     """List controls that mitigate this risk."""
     # Verify risk exists
     result = await db.execute(
         select(Risk).where(Risk.id == risk_id)
     )
     risk = result.scalar_one_or_none()
     if not risk:
         raise HTTPException(status_code=404, detail="Risk not found")
     
-    # Verify department access
-    check_department_access(risk.department_id, current_user)
+    # Allow access via ownership (same pattern as GET /risks/{id})
+    has_access = False
+    if await is_risk_kri_reporting_owner(db, current_user.id, risk_id):
+        has_access = True
+    elif await is_risk_control_owner(db, current_user.id, risk_id):
+        has_access = True
+    else:
+        # Fall back to department check
+        try:
+            check_department_access(risk.department_id, current_user)
+            has_access = True
+        except HTTPException:
+            pass
+    
+    if not has_access:
+        raise HTTPException(status_code=403, detail="Access denied")
```

### Same Pattern for Related Endpoints

Apply same fix to:
- `POST /risks/{id}/controls` - link control to risk
- `DELETE /risks/{id}/controls/{control_id}` - unlink control
