# Plan 152-02: Fix control_id_code Reference (Non-Existent Field)

## Priority: 🔴 CRITICAL (will 500)

## Problem
`approvals.py:123` references `resource.control_id_code` but `Control` model has no such field.

**Affected code:**
```python
# approvals.py line 123
resource_name = f"{resource.control_id_code}: {resource.name[:50] if resource.name else ''}"
```

**Impact:** Creating approval requests for Controls via the generic `/approvals` endpoint will crash with `AttributeError`.

---

## Proposed Changes

### [MODIFY] [approvals.py](../../../backend/app/api/v1/endpoints/approvals.py)

**Line 123:** Use Control ID and name instead of non-existent `control_id_code`
```diff
-        resource_name = f"{resource.control_id_code}: {resource.name[:50] if resource.name else ''}"
+        resource_name = f"Control #{resource.id}: {resource.name[:50] if resource.name else ''}"
```

---

## Investigation Notes

The `Control` model fields (from `control.py`):
- `id` - primary key
- `name` - control name (Mapped[str])
- No `control_id_code` field exists

This is different from `Risk` which has `risk_id_code`.
