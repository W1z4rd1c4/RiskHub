# Plan 152-01: Fix Department head_id → manager_id Mismatch

## Priority: 🔴 CRITICAL (will crash)

## Problem
`approval_helpers.py` references `department.head_id` but `Department` model defines `manager_id`.

**Affected locations:**
- `approval_helpers.py:62-65` - Control approval fallback
- `approval_helpers.py:129-130` - Risk approval fallback
- `risks.py:655-656` - Delete risk approval

**Impact:** Control/Risk edit/delete approvals will crash with `AttributeError: 'Department' object has no attribute 'head_id'`

---

## Proposed Changes

### [MODIFY] [approval_helpers.py](../../../backend/app/core/approval_helpers.py)

**Lines 62-65:** Change `head_id` to `manager_id`
```diff
-    if control.department and control.department.head_id:
-        if requester_id and control.department.head_id == requester_id:
+    if control.department and control.department.manager_id:
+        if requester_id and control.department.manager_id == requester_id:
             return None  # Force escalation to privileged approvers
-        return control.department.head_id
+        return control.department.manager_id
```

**Lines 129-130:** Change `head_id` to `manager_id`
```diff
-    if risk.department and risk.department.head_id and risk.department.head_id != requester_id:
-        return risk.department.head_id
+    if risk.department and risk.department.manager_id and risk.department.manager_id != requester_id:
+        return risk.department.manager_id
```

---

### [MODIFY] [risks.py](../../../backend/app/api/v1/endpoints/risks.py)

**Lines 655-656:** Change `head_id` to `manager_id`
```diff
-        if dept and hasattr(dept, 'head_id') and dept.head_id and dept.head_id != current_user.id:
-            primary_approver_id = dept.head_id
+        if dept and dept.manager_id and dept.manager_id != current_user.id:
+            primary_approver_id = dept.manager_id
```
