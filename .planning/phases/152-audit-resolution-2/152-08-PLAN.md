# Plan 152-08: Fix Control Execution Department Scoping

## Priority: 🟡 MEDIUM (functionality gap)

## Problem
Control owners cannot see executions for their cross-department controls because the executions endpoint is department-scoped only.

**Suspected location:** `executions.py:52`

**Expected behavior:** Control owner should see all executions for controls they own, regardless of control's department.

**Current behavior:** Executions are filtered by user's department only, excluding cross-department owned controls.

---

## Investigation Required

### Step 1: Analyze executions.py
View the execution list endpoint to confirm:
- How executions are filtered
- Whether control owner access is considered
- What department scoping is applied

### Step 2: Identify Fix Pattern
Same pattern as risks/controls:
1. Check if user is control owner
2. If yes, include executions for owned controls
3. Otherwise apply department filter

---

## Proposed Changes

### [MODIFY] [executions.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/executions.py)

Add control owner access to execution list query:
```python
# Import
from app.core.permissions import get_control_ids_where_owner

# In list endpoint
owned_control_ids = await get_control_ids_where_owner(db, current_user.id)
dept_ids = get_user_department_ids(current_user)

if dept_ids is None:
    # Privileged user - see all
    pass
elif owned_control_ids:
    # Include owned controls + department controls
    query = query.where(
        or_(
            ControlExecution.control_id.in_(owned_control_ids),
            Control.department_id.in_(dept_ids)
        )
    )
else:
    # Department only
    query = query.where(Control.department_id.in_(dept_ids))
```
