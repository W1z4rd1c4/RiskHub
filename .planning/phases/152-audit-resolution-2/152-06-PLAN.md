# Plan 152-06: Fix Exception Handling & Commit/Rollback Patterns

## Priority: 🟠 HIGH (session corruption risk)

## Problem
Multiple locations catch exceptions after commits without rollback, potentially leaving DB session in dirty state.

**Affected locations:**

1. **deps.py:82-88** - Commits in `get_current_user`, swallows exception without rollback
```python
try:
    await db.commit()
    await db.refresh(user)
except Exception as e:
    pass  # ❌ No rollback, exception swallowed
```

2. **approvals.py:185-189** - Notification after commit, swallows exception
```python
try:
    await NotificationService.notify_approvers(db, approval)
    await db.commit()
except Exception:
    pass  # ❌ No rollback, notification failure hidden
```

---

## Proposed Changes

### [MODIFY] [deps.py](../../../backend/app/api/deps.py)

**Lines 82-88:** Add rollback and logging
```diff
 try:
     await db.commit()
     await db.refresh(user)
 except Exception as e:
-    pass  # Notification failure should not fail the approval request
+    await db.rollback()  # Ensure clean session state
+    logger.warning(f"Failed to update last_active_at for user {user.id}: {e}")
```

### [MODIFY] [approvals.py](../../../backend/app/api/v1/endpoints/approvals.py)

**Lines 185-189:** Add rollback and logging
```diff
 try:
     await NotificationService.notify_approvers(db, approval)
     await db.commit()
 except Exception:
-    pass  # Notification failure should not fail the approval request
+    await db.rollback()
+    import logging
+    logging.getLogger(__name__).warning(
+        f"Failed to notify approvers for approval #{approval.id}"
+    )
```

Search for other instances of this pattern:

Fix any other locations found.
