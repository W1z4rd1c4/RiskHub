  <name>Task 1: Create Control model with 13-point structure</name>
  <files>backend/app/models/control.py, backend/app/models/__init__.py</files>
  <action>
    Create Control model implementing all 13 points from DEFINICIA KONTROL:
    
    1. name: String(255) - Názov kontroly
    2. description: Text - Stručný popis kontroly
    3. data_source: String(500) - Zdroj dát/vstup
    4. methodology_reference: String(500) - Smernica/Metodický postup/Príkaz GR
    5. control_form: Enum('manual', 'automatic') - Forma kontroly
    6. process_owner_position: String(255) - Pozícia zodpovedná za process
    7. control_owner_id: ForeignKey(users.id) - Pozícia zodpovedná za kontrolu
    8. executor_position: String(255) - Kto vykonáva kontrolu
    9. frequency: Enum('daily', 'weekly', 'monthly', 'quarterly', 'annually', 'ad_hoc')
    10. risk_level: Integer (1-5, 5 is maximum)
    11. output_description: Text - Čo je výstupom kontroly
    12. report_recipient: String(500) - Komu sa výsledok reportuje
    13. documentation_location: String(500) - Kam sa ukladá vykonanie
    
    Additional fields:
    - id: Integer primary key
    - department_id: ForeignKey(departments.id) - owning department
    - status: Enum('draft', 'active', 'inactive', 'archived')
    - created_at, updated_at: timestamps
    - created_by_id, updated_by_id: ForeignKey(users.id) for audit
    
    Use SQLAlchemy 2.0 style with Mapped annotations. Create Enums as Python Enums.
    Update __init__.py to export the new model.
  </action>

<task type="auto">
  <name>Task 5: Create Risk model with Control-Risk linkage</name>
  <files>backend/app/models/risk.py, backend/app/models/__init__.py</files>
  <action>
    Create Risk model based on OS 18 Řízení rizik - Registr rizik structure:
    
    Risk model fields:
    - id: Integer primary key
    - risk_id_code: String(50) - Risk ID (e.g., "Mkt-R01", "UP_NZ_CAT_07_01")
    - process: String(255) - Main process (e.g., "Marketing", "Vývoj produktů")
    - subprocess: String(255) - Subprocess/area
    - risk_type: Enum('strategic', 'operational') - S/O from OS 18
    - category: String(100) - Risk category (Operační riziko, Upisovací riziko, etc.)
    - description: Text - Risk description
    - department_id: ForeignKey(departments.id) - Owner department
    - owner_id: ForeignKey(users.id) - Risk owner/responsible person
    
    Gross Risk (before controls):
    - gross_probability: Integer (1-5)
    - gross_impact: Integer (1-5)
    - gross_score: Integer (computed: probability × impact, 1-25)
    
    Net Risk (after controls applied):
    - net_probability: Integer (1-5)
    - net_impact: Integer (1-5)
    - net_score: Integer (computed: probability × impact, 1-25)
    
    Metadata:
    - status: Enum('active', 'monitoring', 'closed', 'archived')
    - is_priority: Boolean - Whether in Risk Catalog (high priority)
    - kri_indicator: String(500) - Key Risk Indicator description
    - kri_threshold_green: String(255) - Expected range (green)
    - kri_threshold_yellow: String(255) - Elevated risk threshold
    - kri_threshold_red: String(255) - Critical threshold
    - created_at, updated_at: timestamps
    
    Create ControlRiskLink junction table (many-to-many):
    - id: Integer primary key
    - control_id: ForeignKey(controls.id)
    - risk_id: ForeignKey(risks.id)
    - effectiveness: Enum('high', 'medium', 'low') - How effectively control mitigates risk
    - notes: Text - Explanation of how control mitigates risk
    - created_at: timestamp
    
    Add relationships:
    - Risk.controls -> list of Control (via ControlRiskLink)
    - Control.risks -> list of Risk (via ControlRiskLink)
    
    Use SQLAlchemy 2.0 Mapped annotations. Update __init__.py.
  </action>
  <verify>python -c "from app.models.risk import Risk, ControlRiskLink, RiskType, RiskStatus; print('OK')"</verify>
