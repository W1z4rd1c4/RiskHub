## Technical Details ("Option A: Log Forwarding")
Instead of building an API for SIEMs to pull from, we will optimize for **Log Forwarding**. This means the application must emit logs in a format that agents like Filebeat, Fluentd, or Splunk Universal Forwarder can parse immediately without complex regex.

Key Requirements:
1.  **JSON Format**: Every log line must be a valid JSON object.
2.  **Context**: Every log must have `timestamp`, `level`, `request_id`, `user_id` (if auth), `ip`.
3.  **Library**: We will use `structlog` as it's the industry standard for Python structured logging.
