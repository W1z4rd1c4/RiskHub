## Proposed Metrics (CRO Perspective)

### Row 1: Risk Posture (Existing + Enhanced)
| Metric | Source | Direction |
|--------|--------|-----------|
| **New Risks** | Risk.created_at in period | ↑ = Bad |
| **Closed Risks** | Risk.status = closed in period | ↑ = Good |
| **Active Risks** | Risk.status = active | ↑ = Bad |
| **Priority Risks** | Risk.is_priority = True | ↑ = Bad |
| **KRI Breaches** | KRI.breach_status != 'within' | ↑ = Bad |
| **Pending Approvals** | ApprovalRequest.status = pending | ↑ = Attention |

### Row 2: Audit & Control Effectiveness
| Metric | Source | Direction |
|--------|--------|-----------|
| **Audit Activity** | ControlExecution count in period | ↑ = Good |
| **Failed Audits** | ControlExecution.result = failed in period | ↑ = Bad |
| **Control Coverage** | % of Active Risks with ≥1 linked Control | ↑ = Good |
| **Unaudited Controls** | Active Controls with 0 executions in period | ↑ = Bad |

### Row 3: Governance Health
| Metric | Source | Direction |
|--------|--------|-----------|
| **Orphaned Items** | OrphanedItem count (unresolved) | ↑ = Bad |
| **KRI Health** | % KRIs with breach_status = 'within' | ↑ = Good |
| **Overdue KRIs** | KRIs past due date | ↑ = Bad |
| **Activity Volume** | ActivityLog count in period | Neutral |
