<task type="auto">
  <name>Task 2: Update QuestionnaireDeadlineService logic (timezone-safe)</name>
  <action>
Update the deadline/reminder job logic to:
- Read cadence from GlobalConfig keys added in Task 1 (fallback to defaults if missing).
- Send pre-due reminder exactly `N` days before due date (once).
- If overdue, send reminder only on the configured weekday (default Monday), and dedupe so it is at most one per week.
- Recipient is Risk Owner only.

Important implementation details:
- The scheduler runs daily (`backend/app/core/scheduler.py`), so date-based checks must be stable:
  - Use `now = datetime.now(UTC)` and `today = now.date()` (avoid `date.today()` which can differ by server timezone).
- Pre-due check should be date-based (not hour-based): `q.due_at.date() == today + timedelta(days=pre_due_days)`.
- Overdue check should gate on weekday: `q.due_at < now and today.weekday() == overdue_weekday`.
- Keep duplicate prevention via `_check_duplicate_notification`:
  - Pre-due: ensure “once per questionnaire” (lookback window can remain 7 days).
  - Overdue: ensure “once per week” (7-day lookback is acceptable).
  </action>
  <done>Reminders match the intended schedule and do not spam duplicates.</done>
</task>
