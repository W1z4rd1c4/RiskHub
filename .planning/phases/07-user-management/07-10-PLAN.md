# Phase 7.10: Dynamic Role Selection in User Forms

> **Source**: [PERMISSIONAUDIT.MD](file:///Users/stefanlesnak/Antigravity/Risk%20App/.planning/phases/07-user-management/PERMISSIONAUDIT.MD) - High Issue #1, #2

## Objective

Replace hard-coded role IDs in `UserNewPage.tsx` and `UserDetailPage.tsx` with dynamic role fetching from the API. Currently, role IDs are hard-coded, which can assign unintended roles if seed/migrations differ between environments.

## Proposed Changes

### [MODIFY] [users.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/users.py)

Add new endpoint to list available roles:
```python
@router.get("/roles", response_model=List[RoleResponse])
async def list_roles(
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(get_current_user)
):
    """List all available roles for user assignment."""
    result = await db.execute(select(Role))
    return result.scalars().all()
```
