# Plan 07-07: Phase 7 Audit Remediation

## Goal
Address critical and high-severity issues identified in the Phase 7 Audit Report (`AUDIT_REPORT.md`). This includes fixing broken update handlers, securing authentication flows, and resolving race conditions in the frontend state management.

## Context
The Phase 7 audit revealed several blockers for production readiness:
- **API Method Mismatch**: Backend uses `PUT` but implements partial updates; frontend sends `PATCH`.
- **Broken Password Updates**: `update_user` handler fails to hash passwords or map to the correct DB field.
- **Auth Race Conditions**: `AuthContext` lacks abort logic, leading to specific "Phantom Auth" states.
- **Insecure Mock Auth**: Development mock endpoints are not properly isolated.

## Execution Context
- **Source**: `AUDIT_REPORT.md`
- **Focus**: `backend/app/api/v1/endpoints/`, `frontend/src/contexts/`, `frontend/src/services/`

## Tasks

### 1. Backend: Fix API Methods & Schemas
- [ ] **Standardize Update Methods**:
  - Change `risks.py` update endpoint to `PATCH` / `HTTP 200`.
  - Change `controls.py` update endpoint to `PATCH` / `HTTP 200`.
  - Change `users.py` update endpoint to `PATCH` / `HTTP 200` (ensure consistency).
  - Ensure all update schemas (`RiskUpdate`, `ControlUpdate`, `UserUpdate`) are compatible with partial updates (`exclude_unset=True`).
  - Cleanup duplicate fields in `RiskUpdate` schema.

### 2. Backend: Fix User Management Logic
- [ ] **Fix Password Updates**:
  - Update `users.py` `update_user` function:
    - Use `user_data.model_dump(exclude_unset=True)`.
    - Pop `password` from the dict.
    - If `password` is present: `user.hashed_password = get_password_hash(password)`.
    - Apply remaining fields via `setattr`.
- [ ] **Email Uniqueness Check**:
  - Add check in `update_user` to ensure new email doesn't conflict with existing user (if email changed).

### 3. Backend: Security Hardening
- [ ] **Secure Mock Auth**:
  - Update `users.py` to conditionally register `mock_login` endpoint (or return 404) if `MOCK_AUTH_ENABLED` is false.
  - Remove/migrate `get_current_user` in `security.py` to use `deps.get_current_user` logic for JWT in production.
  - Explicitly restrict mock auth logic to development environment.
- [ ] **State Transition Rules**:
  - Add validation in `risks.py` and `controls.py` to prevent transitioning from `archived` to `active`.

### 4. Frontend: Auth & State Fixes
- [ ] **Fix Auth Race Conditions**:
  - Implement `AbortController` in `AuthContext.tsx` `fetchCurrentUser`.
  - Track `isMounted` state to prevent updates after unmount.
- [ ] **Global Error Handling**:
  - Create a shared `apiClient.ts` (wrapper around `fetch` or `axios`) that:
    - Injects `Authorization: Bearer <token>` header.
    - Injects `X-Mock-User-Id` header if configured.
    - Intercepts `401 Unauthorized` responses to trigger a custom event or callback for logout.

### 5. Frontend: Service Standardization
- [ ] **Standardize API Services**:
  - Refactor `userApi.ts`, `riskApi.ts`, `controlApi.ts` to use the new shared `apiClient`.
  - Ensure all services consistently use the same token storage key (`token` vs `access_token`) and header logic.
  - Update `controlApi.updateControl` to use `PATCH`.
