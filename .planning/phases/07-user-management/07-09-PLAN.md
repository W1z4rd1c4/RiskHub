# Phase 7.09: Execution Endpoint Department Scoping

> **Source**: [PERMISSIONAUDIT.MD](file:///Users/stefanlesnak/Antigravity/Risk%20App/.planning/phases/07-user-management/PERMISSIONAUDIT.MD) - Critical Issue #2

## Objective

Add department scoping and permission checks to control execution endpoints. Currently, any authenticated user can read or create executions for any control, bypassing department boundaries.

## Execution Context

- [executions.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/executions.py) - 3 endpoints to secure
- [permissions.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/core/permissions.py) - Permission helpers
- [control.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/models/control.py) - Control model with `department_id`

## Proposed Changes

### [MODIFY] [executions.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/executions.py)

1. Import permission utilities:
   ```python
   from app.core.permissions import get_user_department_ids, check_department_access
   from app.core.security import require_permission
   ```

2. **`read_executions` (GET "")**:
   - Join to Control table
   - Filter by `Control.department_id.in_(dept_ids)` for non-privileged users

3. **`create_execution` (POST "")**:
   - Add `@require_permission("controls", "write")`
   - After fetching control, call `check_department_access(control.department_id, current_user)`

4. **`read_execution` (GET "/{id}")**:
   - After fetching execution, check `check_department_access(execution.control.department_id, current_user)`
