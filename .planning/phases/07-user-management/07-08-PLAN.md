# Phase 7.08: Report Endpoint Department Scoping

> **Source**: [PERMISSIONAUDIT.MD](file:///Users/stefanlesnak/Antigravity/Risk%20App/.planning/phases/07-user-management/PERMISSIONAUDIT.MD) - Critical Issue #1

## Objective

Add department scoping and permission checks to all report export endpoints. Currently, any authenticated user can export all controls/risks across departments by omitting `department_id`.

## Execution Context

- [reports.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/reports.py) - 5 endpoints to secure
- [permissions.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/core/permissions.py) - `get_user_department_ids`, `check_department_access`
- [conftest.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/tests/conftest.py) - Test fixtures for roles

## Proposed Changes

### [MODIFY] [reports.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/reports.py)

1. Import permission utilities:
   ```python
   from app.core.permissions import get_user_department_ids
   from app.core.security import require_permission
   ```

2. Add `require_permission("reports", "read")` decorator to all 5 endpoints:
   - `download_controls_pdf`
   - `download_controls_excel`
   - `download_risks_pdf`
   - `download_risks_excel`
   - `download_summary_pdf`

3. Apply department scoping using the pattern:
   ```python
   dept_ids = get_user_department_ids(current_user)
   if dept_ids is not None:  # None = privileged user
       if not dept_ids:
           return []  # No departments = no data
       query = query.where(Control.department_id.in_(dept_ids))
       # If user-provided department_id, validate it's in their scope
       if department_id and department_id not in dept_ids:
           raise HTTPException(403, "Access denied to this department")
   ```
