# Phase 7.11: KRI Permission Enforcement

> **Source**: [PERMISSIONAUDIT.MD](file:///Users/stefanlesnak/Antigravity/Risk%20App/.planning/phases/07-user-management/PERMISSIONAUDIT.MD) - High Issue #3, Medium Issue #4

## Objective

Add proper permission enforcement to KRI endpoints. Currently, KRI create/update/delete endpoints have no permission enforcement beyond department access. The UI uses `risks:write` but backend has no formal enforcement.

## Decision Required

> [!IMPORTANT]
> **Should KRIs use distinct `kris:*` permissions or inherit from `risks:*`?**
> 
Explicitly tie KRI actions to `risks:*` permissions  
> 
> **Recommendation**: Option B (inherit from risks) - simpler, aligns with current UI behavior

## Execution Context

- [kris.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/kris.py) - Endpoints to secure
- [usePermissions.ts](file:///Users/stefanlesnak/Antigravity/Risk%20App/frontend/src/hooks/usePermissions.ts) - Frontend permission hooks

## Proposed Changes

### [MODIFY] [kris.py](file:///Users/stefanlesnak/Antigravity/Risk%20App/backend/app/api/v1/endpoints/kris.py)

1. Import `require_permission`:
   ```python
   from app.core.security import require_permission
   ```

2. Add permission checks to mutating endpoints:
   - `create_kri`: Add `require_permission("risks", "write")`
   - `update_kri`: Add `require_permission("risks", "write")`  
   - `delete_kri`: Add `require_permission("risks", "delete")`

3. List/read endpoints already have department scoping - add `require_permission("risks", "read")` for consistency

### [MODIFY] [usePermissions.ts](file:///Users/stefanlesnak/Antigravity/Risk%20App/frontend/src/hooks/usePermissions.ts)

Update KRI permission flags to explicitly use risks permissions:
```typescript
// Map KRI actions to risks permissions for clarity
canCreateKRI: hasPermission('risks', 'write'),
canEditKRI: hasPermission('risks', 'write'),
canDeleteKRI: hasPermission('risks', 'delete'),
```

Add comment documenting that KRIs inherit risks permissions.
