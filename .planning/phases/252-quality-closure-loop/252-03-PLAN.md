# Plan 252-03: KRI Form Decomposition

## Objective

Split `frontend/src/components/KRIForm.tsx` into a stable facade plus typed internal modules without changing public behavior.

## Assumptions

- `frontend/src/components/KRIForm.tsx` remains the import-stable public entrypoint.
- Vendor-context behavior and approval-queued behavior must remain identical.

## Steps

1. Create `frontend/src/components/kri-form/`.
2. Move container behavior into `KRIFormContainer.tsx`.
3. Move local form state to `useKriFormState.ts`.
4. Move remote lookups to `useKriLookups.ts`.
5. Move create/update/approval flow to `useKriSubmit.ts`.
6. Split view concerns into step components.
7. Move pure helpers/selectors out of JSX.
8. Remove raw `console.error` from the KRI form path.
9. Extend existing KRI form tests and add helper/hook coverage.

## Wave Gate

- `cd frontend && npm run test:run -- src/components/__tests__/KRIForm.vendor-context.test.tsx src/components/__tests__/KRIModal.vendor-selection.test.tsx src/pages/__tests__/KRIForms.vendor-context.test.tsx src/__tests__/approval_edit_update_handling.spec.ts`
- `cd frontend && npm run lint && npx tsc --noEmit`
