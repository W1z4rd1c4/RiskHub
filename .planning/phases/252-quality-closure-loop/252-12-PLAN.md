# Plan 252-12: Deterministic Migration Script Safety

## Objective

Replace the legacy destructive workbook migration scripts with deterministic import commands that default to dry-run, require explicit input paths, fail closed on unresolved matches, and only allow destructive reset when explicitly requested. For risks specifically, non-reset apply must match existing rows by workbook identity instead of row-order-generated `risk_id_code` values.

## Steps

1. Rewrite `backend/scripts/migrate_risks.py` as a dry-run/apply import command with explicit `--input`, `--report`, and `--allow-reset` contracts.
   - Parse column `F` as `name` and column `G` as `description`.
   - In non-reset mode, match exact normalized `(process, subprocess, name)` and preserve the existing `risk_id_code`.
   - In non-reset mode, create unmatched rows using the canonical `generate_risk_id_code(...)` helper.
2. Rewrite `backend/scripts/migrate_controls.py` to upsert by control name, rebuild risk links deterministically, and avoid global deletes unless `--allow-reset` is passed.
3. Rewrite `backend/scripts/migrate_kris.py` to upsert by `(risk_id, metric_name)`, remove the random fallback, and fail closed on unmatched rows.
4. Add focused backend regression tests covering dry-run non-destructiveness, risk identity preservation under row reordering, new-row insertion without renumbering existing risks, ambiguity failure, duplicate workbook-key failure, and fail-closed unmatched KRI handling.
5. Update backend script docs and Phase 252 state once the new contract is green.
