# Plan 252-04: Vendor Form Decomposition

## Objective

Split `frontend/src/components/VendorForm.tsx` into a stable facade plus typed internal modules without changing current behavior.

## Assumptions

- `frontend/src/components/VendorForm.tsx` remains the public entrypoint.
- Existing process/subprocess suggestions and owner-to-department autofill remain unchanged.

## Steps

1. Create `frontend/src/components/vendor-form/`.
2. Move container behavior to `VendorFormContainer.tsx`.
3. Extract lookup loading into `useVendorLookups.ts`.
4. Extract local state into `useVendorFormState.ts`.
5. Extract payload construction and submit flow into `useVendorSubmit.ts` plus `vendorForm.mappers.ts`.
6. Split UI sections by business concept.
7. Remove raw `console.error` from the Vendor form path.
8. Add new Vendor form tests because coverage does not currently exist.

## Wave Gate

- `cd frontend && npm run test:run -- src/components/__tests__/VendorForm.test.tsx src/components/__tests__/VendorForm.payloads.test.ts`
- `cd frontend && npm run lint && npx tsc --noEmit`
