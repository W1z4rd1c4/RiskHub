# Plan 252-07: Service Facade Cleanup

## Objective

Split `apiClient` and `adminApi` internally without redesigning the auth/session stack and while preserving public imports.

## Assumptions

- `@/services/apiClient` and `@/services/adminApi` remain stable imports.
- `AuthContext`, `useAuthActions`, `useAuthBootstrap`, session stores, and SSO flow remain untouched unless needed for compilation/test repair.

## Steps

1. Create `frontend/src/services/api/` internal modules.
2. Turn `frontend/src/services/apiClient.ts` into a thin public assembler/facade.
3. Create `frontend/src/services/admin/` bounded modules.
4. Turn `frontend/src/services/adminApi.ts` into a thin public assembler/facade.
5. Preserve `401` retry, `getBlob`, `toUiMessageKey`, and current admin surface behavior exactly.
6. Extend existing service/admin tests and add focused new service-unit coverage.

## Wave Gate

- `cd frontend && npm run test:run -- src/services/__tests__/apiClient.401-recovery.test.ts src/services/__tests__/apiClient.errors.test.ts src/services/__tests__/apiClient.requestBuilder.test.ts src/pages/admin-console/__tests__/AdminConsoleOpsPanels.outbox.test.tsx src/pages/__tests__/UsersPage.sso-cta.test.tsx`
- `cd frontend && npm run lint && npx tsc --noEmit`
