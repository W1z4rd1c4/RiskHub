# Plan 252-06: Dashboard Page Split

## Objective

Reduce `frontend/src/pages/DashboardPage.tsx` to a route facade while preserving existing query and widget behavior.

## Assumptions

- Existing widgets and the aggregate overview query remain in place.
- Committee view switching and permissions remain unchanged.

## Steps

1. Create `frontend/src/pages/dashboard/`.
2. Move container behavior to `DashboardPageContainer.tsx`.
3. Extract stat-card construction to `dashboardStats.ts`.
4. Extract export/drilldown/navigation helpers to `dashboardNavigation.ts`.
5. Extend existing dashboard overview tests and add `dashboardStats.test.ts`.
