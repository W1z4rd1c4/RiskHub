# Plan 252-02: Gate Ratchet Before Frontend Refactors

## Objective

Raise the phase-relevant static quality floor before the large frontend refactors land.

## Assumptions

- Repo-wide frontend TS-safety enablement is feasible because the measured baseline is small and known.
- Backend lint tightening stays touched-file scoped; repo-wide Ruff expansion is out of scope.

## Steps

1. Enable blocking frontend ESLint rules:
   - `@typescript-eslint/no-unsafe-argument`
   - `@typescript-eslint/no-unsafe-assignment`
   - `@typescript-eslint/no-unsafe-return`
   - `@typescript-eslint/prefer-promise-reject-errors`
   - `@typescript-eslint/require-await`
2. Fix the known baseline files listed in the phase plan.
3. Add `mypy` to backend dev dependencies and create `backend/mypy.ini`.
4. Add blocking mypy CI for Phase 252 backend files only.
5. Add touched-file Ruff CI for Phase 252 backend files with `--select UP,SIM`.
6. Measure backend/frontend current total coverage and set blocking initial thresholds to `floor(current measured total)`.
7. Add phase-owned ESLint overrides for `no-console`, `max-lines`, `max-lines-per-function`, and `complexity`.

## Wave Gate

- `cd frontend && npm run lint && npx tsc --noEmit && npm run test:coverage`
- `cd backend && ./venv/bin/mypy app/core/activity_logger.py app/core/activity_redaction.py app/bootstrap_runtime.py app/bootstrap_validation.py app/core/settings && ./venv/bin/ruff check app/core/activity_logger.py app/core/activity_redaction.py app/bootstrap_runtime.py app/bootstrap_validation.py app/core/settings --select UP,SIM`
- `make -f scripts/Makefile test`
