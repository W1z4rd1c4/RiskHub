## Findings Summary

1. Access write endpoint only strictly protects `role_id`/`access_scope` changes, leaving `department_id` and `manager_id` mutable by global non-admin users.
2. Frontend sends/edits `log_rotation_size_mb` and `log_retention_count`, while backend requires app/audit split fields.
3. Canonical admin model is platform-scoped explicit permissions; legacy script and test fixtures still encode wildcard admin assumptions.

## Recommended Decomposition

- Wave 1: close backend auth gap + log-config contract backend shims.
- Wave 2: align frontend behavior and seed/fixture/test contract.
- Wave 3: run regression matrix and reconcile docs.

## Implementation Constraints

- Keep backend enforcement as authority.
- Avoid introducing behavior regressions for CRO-managed Risk Hub paths.
- Preserve API compatibility for one transition cycle for log config payloads.
