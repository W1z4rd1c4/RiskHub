# Summary 156.1-02: Frontend Access Edit Gating Alignment

## Outcome
- Split frontend access permissions into read visibility vs edit mutation capability.
- Removed UI pathways that advertise unauthorized access-user edits.

- Updated authz model in `frontend/src/authz/policy.ts` with `canEditAccessUsers = isPlatformAdmin || isCRO`.
- Exposed new capability in `frontend/src/hooks/usePermissions.ts`.

  - `frontend/src/components/access/UsersTable.tsx`
  - `frontend/src/components/access/AccessEditModal.tsx`
- Defensive modal submit block now prevents unauthorized mutation attempts and shows local error.
- Added regression coverage in `frontend/src/pages/__tests__/rbac_gating.test.tsx`:
  - Global non-admin can view access data but cannot see edit action.
  - Admin can see edit action.
