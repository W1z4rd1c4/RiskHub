  <action>
Add issue-focused dashboard metrics with scope filters:
- `GET /dashboard/issues-summary`:
  - open_issues
  - overdue_issues
  - high_severity_open
  - median_days_open
- `GET /dashboard/issues-aging`:
  - aging buckets (0-7, 8-30, 31-60, 61+)
- `GET /dashboard/issues-by-severity`:
  - counts by severity for open statuses

Implementation rules:
- Enforce `issues:read` permission on these endpoints (do not piggyback only on `risks:read`).
- Respect department scoping and cross-department ownership exceptions consistent with `Issue` visibility rules.
- Use deterministic date math with UTC normalization.
- Add response schemas in `dashboard.py` schema module.

Metric definitions (explicit):
- “open issues” means `Issue.status != closed` (or equivalent enum) AND not suppressed by an approved, unexpired exception.
- “overdue issues” means `due_at < now` AND open AND not suppressed by exception.
- “high severity open” counts severities in (`high`, `critical`) and open.
- “median_days_open” computed from `opened_at` to now for open issues only, using integer day bucketing.
