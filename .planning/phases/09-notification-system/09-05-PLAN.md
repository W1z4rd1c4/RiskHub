<task type="checkpoint:decision" gate="blocking">
  <decision>Select background task scheduler approach</decision>
  <context>We need a way to run daily KRI deadline checks. The roadmap mentions APScheduler vs Celery.</context>
  <options>
    <option id="apscheduler">
      <name>APScheduler (in-process)</name>
      <pros>Simple setup, no external dependencies, runs in main process, good for single-instance deployments</pros>
      <cons>Won't scale to multiple instances without extra config, no task queue persistence</cons>
    </option>
    <option id="celery">
      <name>Celery with Redis</name>
      <pros>Production-grade, scales well, persistent task queue, beat for scheduling</pros>
      <cons>Requires Redis, more complex setup, overkill for simple daily jobs</cons>
    </option>
    <option id="startup-cron">
      <name>FastAPI startup + asyncio scheduler</name>
      <pros>Zero dependencies, uses Python asyncio, integrates with FastAPI lifecycle</pros>
      <cons>Limited features, no persistence, restarts lose schedule</cons>
    </option>
  </options>
  <resume-signal>Select: apscheduler, celery, or startup-cron</resume-signal>
</task>

  <action>
    Create service with methods:
    
    ```python
    class KRIDeadlineService:
        @staticmethod
        async def check_kri_deadlines(db: AsyncSession) -> dict[str, int]:
            """
            Check all KRIs and generate notifications:
            1. Due in 7 days → KRI_DUE_SOON (if not already notified)
            2. Due tomorrow → KRI_DUE_TOMORROW (if not already notified)
            3. Overdue → KRI_OVERDUE (if not already notified for this period)
            4. Near breach (value >= 80% of limit) → KRI_NEAR_BREACH
            
            Returns counts: {"due_soon": N, "due_tomorrow": M, ...}
            """
            
        @staticmethod
        async def _check_duplicate_notification(
            db: AsyncSession,
            kri_id: int,
            notification_type: NotificationType,
            lookback_days: int = 7,
        ) -> bool:
            """Prevent spamming same notification within lookback period"""
    ```
    
    Query KeyRiskIndicator table for:
    - KRIs with reporting_deadline within 7 days
    - KRIs with reporting_deadline tomorrow
    - KRIs past reporting_deadline
    - KRIs with current_value approaching limit_value (80%+)
    
    Use NotificationService.create_notification() for each.
    Notify KRI owner (kri.responsible_id or fallback to department head).
    Also notify Risk Managers for overdue/near-breach.
  </action>

<task type="auto">
  <name>Task 3: Set up scheduler based on chosen approach</name>
  <files>backend/app/core/scheduler.py, backend/app/main.py</files>
  <action>
    Based on decision from Task 1:
    
    **If APScheduler:**
    ```python
    # scheduler.py
    from apscheduler.schedulers.asyncio import AsyncIOScheduler
    
    scheduler = AsyncIOScheduler()
    
    async def run_kri_check():
        async with get_db_context() as db:
            result = await KRIDeadlineService.check_kri_deadlines(db)
            logging.info(f"KRI deadline check: {result}")
    
    scheduler.add_job(run_kri_check, 'cron', hour=8, minute=0)  # Daily at 8am
    
    # main.py
    @app.on_event("startup")
    async def startup():
        scheduler.start()
    
    @app.on_event("shutdown")
    async def shutdown():
        scheduler.shutdown()
    ```
    
    **If startup-cron (no external deps):**
    ```python
    # scheduler.py
    import asyncio
    
    async def schedule_kri_checks():
        while True:
            await asyncio.sleep(86400)  # 24 hours
            await run_kri_check()
    
    # main.py - start as background task
    ```
    
    Install required package if APScheduler: `pip install apscheduler`
    Add to requirements.txt.
  </action>
  <verify>Backend starts and logs scheduler initialization</verify>
