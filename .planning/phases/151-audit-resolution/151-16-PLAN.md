# Plan 151-16: Approval Request DB-level Constraints

## Objective
Enforce business rules for pending approvals at the database level to prevent duplicate requests for the same action on the same resource.

## Execution Context
- **Models**: `backend/app/models/approval_request.py`
- **Endpoints**: `backend/app/api/v1/endpoints/risks.py`, `backend/app/api/v1/endpoints/controls.py`
- **Migrations**: Alembic

## Context
Currently, the "duplicate check" happens only in the application code (`if existing.scalar_one_or_none()`). Concurrent requests can bypass this, leading to multiple pending approvals for the same item.

## Tasks

### 1. Database Migration
- [ ] Create a new Alembic migration.
- [ ] Add a unique constraint (Index) to `approval_requests` table.
- [ ] **SQL**: `CREATE UNIQUE INDEX ux_approval_pending ON approval_requests (resource_type, resource_id, action_type) WHERE status = 'PENDING';`
- [ ] Note: Ensure compatibility with both Postgres and SQLite (SQLite 3.9+ supports partial indexes).

### 2. Update Model Definition
- [ ] Update `ApprovalRequest` in `backend/app/models/approval_request.py`.
- [ ] Add the `Index` to `__table_args__` with the `sqlite_where` and `postgresql_where` clauses.

### 3. Update Endpoints to Handle Constraint
- [ ] Update `update_risk` and `delete_risk` in `backend/app/api/v1/endpoints/risks.py`.
- [ ] Update `update_control` and `delete_control` in `backend/app/api/v1/endpoints/controls.py`.
- [ ] Wrap the `db.commit()` in a try/except:
    - Catch `IntegrityError`.
    - Extract details to confirm it's the `ux_approval_pending` violation.
    - Return `HTTP 409 Conflict` with a clear message: "An approval request is already pending for this action."

## Verification
- [ ] **Manual**: Attempt to double-click "Delete" or "Save" (on sensitive field) in the UI and ensure only one request is created and the second returns an error.
- [ ] **Automated**: `test_duplicate_approval_prevention` in a new test file.

## Success Criteria
- [ ] Database contains no duplicate `PENDING` approval requests for the same (type, id, action).
- [ ] API returns 409 instead of 500 when a collision occurs.
