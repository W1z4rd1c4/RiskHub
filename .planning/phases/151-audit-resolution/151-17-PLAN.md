# Plan 151-17: Sensitive Field Detection & Approval Logic Refinement

## Objective
Fix the bug where clearing sensitive fields (setting to `None`) bypasses approval, and improve primary approver selection.

## Problem Statement

### Bug 1: Null Values Bypass Approval
In `backend/app/core/permissions.py` line 237:
```python
if new_val is not None and old_val != new_val:  # BUG: skips when new_val is None
```

**Attack**: User sets `owner_id: null` → No approval triggered → Risk loses oversight.

### Bug 2: Missing Department Handling
In `backend/app/core/approval_helpers.py` line 54:
```python
if control.department and control.department.head_id:  # OK, but no self-approval check
```

## Tasks

### 1. Fix `has_sensitive_field_changes` in `permissions.py`

**File**: `backend/app/core/permissions.py` (lines 212-247)

**Replace the function with:**
```python
def has_sensitive_field_changes(
    resource_type: str, 
    old_data: dict, 
    new_data: dict
) -> tuple[bool, dict]:
    """Check if any sensitive fields are being changed, including clearing to None."""
    sensitive = SENSITIVE_FIELDS.get(resource_type, set())
    changed = {}
    NOT_PROVIDED = object()  # Sentinel to detect "not in payload"
    
    for field in sensitive:
        new_val = new_data.get(field, NOT_PROVIDED)
        if new_val is NOT_PROVIDED:  # Field not in update payload
            continue
        
        old_val = old_data.get(field)
        if old_val == new_val:  # No actual change
            continue
        
        # is_priority: only true→false requires approval (downgrade)
        if field == "is_priority":
            if old_val is True and new_val is False:
                changed[field] = {"old": old_val, "new": new_val}
            continue
        
        # All other sensitive fields: ANY change requires approval
        # Including owner_id: 5→None (clearing owner)
        changed[field] = {"old": old_val, "new": new_val}
    
    return bool(changed), changed
```

**Add new helper for risks:**
```python
async def get_primary_approver_for_risk(
    db: AsyncSession, risk_id: int, requester_id: Optional[int] = None
) -> Optional[int]:
    """Get primary approver for a Risk edit, preventing self-approval."""
    risk_result = await db.execute(
        select(Risk).options(selectinload(Risk.department)).where(Risk.id == risk_id)
    )
    risk = risk_result.scalar_one_or_none()
    if not risk:
        return None
    
    if risk.owner_id and risk.owner_id != requester_id:
        return risk.owner_id
    
    if risk.department and risk.department.head_id != requester_id:
        return risk.department.head_id
    
    return None
```
