# Plan 151-17: Sensitive Field Detection & Approval Logic Refinement

## Objective
Refine the sensitivity detection logic to catch unassigning fields (setting to `None`) and improve the primary approver selection to handle edge cases.

## Execution Context
- **Files**: `backend/app/core/permissions.py`, `backend/app/core/approval_helpers.py`

## Context
1. `has_sensitive_field_changes` currently skips checks if `new_val` is `None`. This means clearing an owner or department bypasses approvals.
2. The `is_priority` special case only flags true→false. We should ensure this is strictly what the business wants or if both should be flagged. (Business usually wants to control "deprioritization" more strictly).
3. `get_primary_approver_for_control` might fail if no owner exists anywhere.

## Tasks

### 1. Update `has_sensitive_field_changes`
- [ ] Modify `backend/app/core/permissions.py`.
- [ ] Change the condition: check if the key exists in `new_data` (payload) rather than checking if it's not `None`.
- [ ] Ensure `old_val != new_val` still holds.
- [ ] Final logic should catch: `owner_id: 1 -> null`.

### 2. Refine `is_priority` Logic
- [ ] Confirm `true -> false` requires approval.
- [ ] Confirm `false -> true` does NOT.
- [ ] Add explicit comments explaining this "one-way gate" logic.

### 3. Improve Primary Approver Selection
- [ ] Update `get_primary_approver_for_control` in `backend/app/core/approval_helpers.py`.
- [ ] Handle cases where `control.department` is None.
- [ ] Handle cases where the requester is the same as the selected primary approver (it should likely escalate or require a second pair of eyes, but the model currently supports privileged approval for high-risk items).

## Verification
- [ ] **Unit Tests**: Add tests to `backend/tests/test_permissions.py` covering `has_sensitive_field_changes` with `None` values.
- [ ] **Integration Tests**: Verify that clearing a risk owner via API triggers an approval request.

## Success Criteria
- [ ] Clearing sensitive fields is correctly caught and routed to approval.
- [ ] Approver selection is robust against missing data (unassigned departments/owners).
