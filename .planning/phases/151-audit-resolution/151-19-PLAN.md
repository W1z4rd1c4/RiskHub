**Issue**: Users cannot cancel their requests once they've been partially approved and moved to `PENDING_PRIVILEGED`. This is inconsistent—the requester should be able to cancel at any pending stage.

**CURRENT CODE (line 638-639):**
```python
if approval.status != ApprovalStatus.PENDING:
```

**REPLACE WITH:**
```python
if approval.status not in (ApprovalStatus.PENDING, ApprovalStatus.PENDING_PRIVILEGED):
```

### 3. Add Tiered Approval Fields to Delete Requests (M-1)

**File**: `backend/app/api/v1/endpoints/risks.py` (lines 580-587)

**Issue**: Delete requests don't set `primary_approver_id` or `requires_privileged_approval`, so the tiered approval flow doesn't work.

**CURRENT CODE:**
```python
approval = ApprovalRequest(
    resource_type=ApprovalResourceType.RISK,
    resource_id=risk.id,
    resource_name=risk.name,
    requested_by_id=current_user.id,
    reason=f"{reason}\n\nDescription: {desc_snippet}" if desc_snippet else reason,
    status=ApprovalStatus.PENDING,
)
```

**REPLACE WITH:**
```python
from app.core.permissions import is_high_risk_for_approval_async

# Determine primary approver and if privileged approval is needed
primary_approver_id = risk.owner_id if risk.owner_id != current_user.id else None
requires_privileged = await is_high_risk_for_approval_async(risk, db)

# Fallback to department head if no owner or self-approval
if not primary_approver_id and risk.department:
    from sqlalchemy import select
    from app.models import Department
    dept_result = await db.execute(select(Department).where(Department.id == risk.department_id))
    dept = dept_result.scalar_one_or_none()
    if dept and dept.head_id and dept.head_id != current_user.id:
        primary_approver_id = dept.head_id

approval = ApprovalRequest(
    resource_type=ApprovalResourceType.RISK,
    resource_id=risk.id,
    resource_name=risk.name,
    requested_by_id=current_user.id,
    reason=f"{reason}\n\nDescription: {desc_snippet}" if desc_snippet else reason,
    status=ApprovalStatus.PENDING,
    action_type=ApprovalActionType.DELETE,
    primary_approver_id=primary_approver_id,
    requires_privileged_approval=requires_privileged,
)
```
