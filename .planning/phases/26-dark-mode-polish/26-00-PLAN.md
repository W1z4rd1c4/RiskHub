<objective>
Polish dark mode theme across all charts and visualizations for consistent appearance in all three themes (RiskHub, Dark, Light).

Purpose: Currently, Recharts components have hardcoded dark backgrounds (`rgba(15, 23, 42, ...)`) in tooltips and inline styles that don't adapt to the selected theme. This creates visual inconsistency, especially in Light theme where dark tooltips clash with the light UI.

Output:
- Theme-aware chart hook (`useChartTheme`) providing colors for current theme
- Updated chart components using theme-aware colors
- Consistent contrast ratios across all themes (WCAG AA compliance)
</objective>

<task type="auto">
  <name>Task 1: Create useChartTheme hook</name>
  <files>frontend/src/hooks/useChartTheme.ts</files>
  <action>
Create a new hook that provides theme-aware colors for Recharts components. The hook should:

1. Import and use `useTheme()` from `@/contexts/ThemeContext`
2. Return an object with chart-specific colors that vary by theme:
   - `tooltipBackground`: Background for chart tooltips
   - `tooltipBorder`: Border for chart tooltips  
   - `tooltipTextPrimary`: Primary text color in tooltips
   - `tooltipTextSecondary`: Secondary/label text color in tooltips
   - `gridStroke`: CartesianGrid stroke color
   - `axisTickFill`: XAxis/YAxis tick text color
   - `activeDotFill`: Active dot fill color (matches background for hollow effect)
   - `cursorStroke`: Tooltip cursor line color

Color values by theme:
- **RiskHub (default)**: Keep current dark values - `rgba(15, 23, 42, 0.9)` background, white text
- **Dark (OLED)**: Pure black - `rgba(10, 10, 10, 0.95)` background, `#f1f1f1` text
- **Light**: White/light gray - `rgba(255, 255, 255, 0.98)` background, `#1e293b` text

Export the hook and a `ChartTheme` type for TypeScript.
  </action>
  <verify>TypeScript builds without errors: `cd frontend && npm run build`</verify>
  <done>Hook exists at `frontend/src/hooks/useChartTheme.ts`, exports `useChartTheme` and `ChartTheme` type</done>
</task>

<task type="auto">
  <name>Task 2: Update Recharts dashboard components</name>
  <files>
    frontend/src/components/dashboard/RiskTrendChart.tsx,
    frontend/src/components/dashboard/KRIBreachHistoryChart.tsx,
    frontend/src/components/dashboard/ControlTrendChart.tsx,
    frontend/src/components/dashboard/CategoryBreakdownCharts.tsx
  </files>
  <action>
Update each chart component to use `useChartTheme()` instead of hardcoded colors:

1. Import `useChartTheme` at the top of each file
2. Call `const chartTheme = useChartTheme()` inside the component
3. Replace hardcoded `contentStyle` in `<Tooltip>`:
   - `backgroundColor` → `chartTheme.tooltipBackground`
   - `border` → `1px solid ${chartTheme.tooltipBorder}`
4. Replace hardcoded `itemStyle` and `labelStyle`:
   - `color` in itemStyle → `chartTheme.tooltipTextPrimary`
   - `color` in labelStyle → `chartTheme.tooltipTextSecondary`
5. Replace hardcoded `CartesianGrid` stroke → `chartTheme.gridStroke`
6. Replace hardcoded `XAxis`/`YAxis` tick fill → `chartTheme.axisTickFill`
7. Replace hardcoded `activeDot` fill → `chartTheme.activeDotFill`
8. Replace hardcoded `cursor` stroke (in Tooltip) → `chartTheme.cursorStroke`

For `CategoryBreakdownCharts.tsx` (pie charts), also update the tooltip styling in `MiniPieChart`.

Do NOT change:
- Gradient colors (these are accent/brand colors that should stay consistent)
- Chart line/area stroke colors (e.g., `#1E84FF`, `#F43F5E`)
- Legend styling
  </action>
  <verify>TypeScript builds: `cd frontend && npm run build`. No errors.</verify>
  <done>All 4 dashboard chart components use `useChartTheme()` for tooltip and grid styling</done>
