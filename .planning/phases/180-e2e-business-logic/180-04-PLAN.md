  <action>
    Create tests for BUSINESS_LOGIC.md §5.1 Approval Status Flow:
    
    1. **PENDING → APPROVED**:
       - Login as Employee, delete a Risk (creates PENDING request)
       - Verify request appears in Approvals list with status "Pending"
       - Login as Risk Owner (primary approver), approve request
       - Verify status changes to "Approved" (if no privileged needed) or "Pending Privileged"
    
    2. **PENDING → REJECTED**:
       - Login as Employee, delete a Risk
       - Login as approver, reject request with reason
       - Verify status is "Rejected", Risk is NOT archived
    
    3. **PENDING → CANCELLED**:
       - Login as Employee, delete a Risk
       - Navigate to Approvals, click Cancel on own request
       - Verify status is "Cancelled", Risk is NOT archived
    
    4. **PENDING_PRIVILEGED → APPROVED**:
       - Login as Employee, edit a high-priority Risk
       - Approve at primary level
       - Verify status is "Pending Privileged"
       - Login as CRO, approve at privileged level
       - Verify status is "Approved", changes are applied
  </action>
