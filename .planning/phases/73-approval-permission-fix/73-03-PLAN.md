# Plan 73-03: Risk Hub UI Validation & Warnings

> **Scope**: ~1.5 hours
> **Depends on**: Plan 73-01 (permission exists)
> **Output**: Risk Hub shows warning if configured approver lacks permission

---

## Objective

Add validation in Risk Hub UI to warn CRO if a configured approver role doesn't actually have the `approvals:write` permission, preventing configuration confusion.

---

## Context

The Risk Hub "Approval Rules" tab allows CRO to configure `approver_roles` for each scenario. However, if Plan 73-02 is not implemented, these roles also need `approvals:write` in the RBAC system.

This plan adds a visual warning when there's a mismatch.

---

## Tasks

### 1. Backend: Add Permission Check Endpoint

**File**: `backend/app/api/v1/endpoints/riskhub.py`

Add new endpoint:

```python
@router.get("/approval-validation")
async def validate_approval_config(
    db: AsyncSession = Depends(get_db),
    current_user: User = Depends(deps.get_current_user_cro),
):
    """
    Validate that configured approver roles have approvals:write permission.
    Returns list of roles missing the required permission.
    """
    # Get all configured approver roles from scenarios
    scenarios = await db.execute(select(ApprovalScenario))
    all_approver_roles = set()
    for s in scenarios.scalars():
        all_approver_roles.update(s.get_approver_roles())
    
    # Check which roles have approvals:write or *:*
    missing_permission = []
    for role_name in all_approver_roles:
        role_result = await db.execute(
            select(Role).where(Role.name == role_name)
        )
        role = role_result.scalar_one_or_none()
        if not role:
            missing_permission.append(role_name)
            continue
        
        # Check for approvals:write or *:*
        has_perm = await db.execute(
            select(RolePermission)
            .join(Permission)
            .where(
                RolePermission.role_id == role.id,
                or_(
                    and_(Permission.resource == "approvals", Permission.action == "write"),
                    and_(Permission.resource == "*", Permission.action == "*"),
                )
            )
        )
        if not has_perm.scalar_one_or_none():
            missing_permission.append(role_name)
    
    return {
        "configured_roles": list(all_approver_roles),
        "missing_permission": missing_permission,
        "valid": len(missing_permission) == 0,
    }
```

### 2. Frontend: Add Warning Banner

**File**: `frontend/src/components/riskhub/ApprovalScenariosPanel.tsx`

```tsx
// Add state for validation
const [validationWarning, setValidationWarning] = useState<string[]>([]);

// Fetch validation on mount
useEffect(() => {
    riskHubApi.getApprovalValidation().then(data => {
        if (!data.valid) {
            setValidationWarning(data.missing_permission);
        }
    });
}, []);

// Render warning banner if issues found
{validationWarning.length > 0 && (
    <div className="bg-amber-500/10 border border-amber-500/30 rounded-lg p-4 mb-4">
        <div className="flex items-center gap-2 text-amber-400">
            <AlertTriangle className="h-5 w-5" />
            <span className="font-medium">Configuration Warning</span>
        </div>
        <p className="text-sm text-slate-300 mt-2">
            The following roles are configured as approvers but lack the required 
            <code className="mx-1 px-1 bg-slate-700 rounded">approvals:write</code> 
            permission: {validationWarning.join(', ')}
        </p>
    </div>
)}
```

async getApprovalValidation(): Promise<{
