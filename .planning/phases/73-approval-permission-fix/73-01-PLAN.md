# Plan 73-01: Permission Infrastructure Fix

> **Scope**: ~45 minutes
> **Depends on**: None
> **Output**: `approvals:write` permission granted to privileged roles

---

## Objective

Create the `approvals:write` permission and grant it to all privileged roles that should be able to approve/reject approval requests.

---

## Context

The `can_resolve_approvals()` function in `app/core/permissions.py` requires:

```python
is_privileged_user(user) and has_permission(user, "approvals", "write")
```

Currently:

- `approvals:write` permission does **not exist** in the `permissions` table
- Only `admin` and `cro` have `*:*` wildcard which matches any permission
- `risk_manager`, `compliance`, `legal`, `internal_audit`, `actuarial` lack this permission

---

## Tasks

### 1. Create Alembic Migration

**File**: `backend/alembic/versions/xxxx_add_approvals_write_permission.py`

```python
def upgrade():
    # 1. Insert 'approvals:write' permission if missing
    op.execute("""
        INSERT INTO permissions (resource, action, description)
        VALUES ('approvals', 'write', 'Approve or reject approval requests')
        ON CONFLICT (resource, action) DO NOTHING;
    """)
    
    # 2. Grant to privileged governance roles
    op.execute("""
        INSERT INTO role_permissions (role_id, permission_id)
        SELECT r.id, p.id
        FROM roles r, permissions p
        WHERE r.name IN ('risk_manager', 'compliance', 'legal', 'internal_audit', 'actuarial')
          AND p.resource = 'approvals' AND p.action = 'write'
        ON CONFLICT DO NOTHING;
    """)
```

### 2. Update Seed Script

**File**: `backend/scripts/seed_roles_permissions.py`

Add to `permissions_data`:

```python
("approvals", "write", "Approve or reject approval requests"),
```

Grant to privileged roles (lines ~88-104).

### 3. Run Migration

```bash
cd backend && source venv/bin/activate
alembic revision --autogenerate -m "add_approvals_write_permission"
# Edit the generated file with the SQL above
alembic upgrade head
```
