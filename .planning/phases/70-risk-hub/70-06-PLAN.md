# 70-06: CRO Role & Department Management in Risk Hub

## Objective

Enable the CRO to manage platform roles AND departments through the Risk Hub:
- **Roles**: Create/edit/delete roles with individual permission checkboxes
- **Departments**: Create/edit/delete departments with hierarchy and managers

This adds two new tabs to Risk Hub: **Roles** and **Departments**.

## Context

### Current State
- Roles are seeded from `RoleType` enum (static, no CRO management)
- Departments exist but are only manageable in separate Departments page
- CRO has Risk Hub access for risk types, config, and approval scenarios

### User Vision
- **Full role builder**: CRO picks individual permissions like LEGO blocks (risks:read, controls:write, etc.)
- **Department management**: CRO manages organizational structure from Risk Hub

### Design Decisions
1. **System roles protected**: Built-in roles (admin, cro, employee) cannot be deleted
2. **Soft delete**: Roles/Departments marked inactive, not hard deleted
3. **Block delete if in use**: Deletion blocked if users assigned
4. **CRO-only access**: Only CRO role can manage (not Admin)

## Tasks

### Backend - Roles (3 tasks)

#### Task 1: Role Management API Endpoints
**File:** `backend/app/api/v1/endpoints/riskhub.py`

Add CRUD endpoints for roles:
```
GET    /riskhub/roles              # List all roles (include_inactive param)
POST   /riskhub/roles              # Create new role
PATCH  /riskhub/roles/{id}         # Update role (name, permissions)
DELETE /riskhub/roles/{id}         # Soft-delete role
POST   /riskhub/roles/{id}/restore # Restore deleted role
GET    /riskhub/permissions        # List all available permissions (for checkbox UI)
```

#### Task 2: Role Schemas
**File:** `backend/app/schemas/riskhub.py`

Add Pydantic schemas:
- `RoleHubRead`: id, name, display_name, description, is_system, is_active, user_count, permissions[]
- `RoleHubCreate`: name, display_name, description, permission_ids[]
- `RoleHubUpdate`: display_name, description, permission_ids[]
- `PermissionRead`: id, resource, action, description

#### Task 3: Role Model Updates + Migration
**File:** `backend/app/models/role.py`

Add to Role model:
- `is_system: bool = False` (protects seeded roles)
- `is_active: bool = True` (soft delete)

**Migration:** Add columns with defaults, mark existing seeded roles as `is_system=True`.
