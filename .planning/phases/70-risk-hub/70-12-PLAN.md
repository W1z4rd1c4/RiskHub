# Plan 70-12: Frontend Configuration & UI Adaptation

## Goal
Expose the new backend configurations to the System Admin in the Risk Hub and adapt the User Interface to respect these settings (e.g., hiding buttons, showing errors).

## Technical Guidance
The frontend must react to the global configuration state. We will use the `useQuery` hook for `globalConfig` to determine feature availability.

**Key Components to Touch**:
1.  **SystemSettingsPanel**: The "Control Panel" for these toggles.
2.  **ControlDetails**: The consumer of the "Control Logging" flag.
3.  **RecordValueModal**: The consumer of the "Strict KRI" flag.

## Proposed Changes

### Frontend Code

#### 1. Admin UI: System Settings
**File:** `frontend/src/components/riskhub/SystemSettingsPanel.tsx`
**Action:**
- Add a new section "Feature Configuration" (or inside "General").
- Add `<Switch>` components for:
    - `"Strict KRI Period Submission"` (binds to `kri_strict_period_submission`)
    - `"Control Execution Logging"` (binds to `control_execution_logging`)
- Ensure `onSave` updates these keys via `riskHubApi.updateConfig`.

#### 2. Feature UI: Control Details
**File:** `frontend/src/components/controls/ControlDetails.tsx` (or `ControlHeader.tsx`)
**Action:**
- Fetch config: `const { data: config } = useQuery(['globalConfig'], ...)`
- **Logic**:
  ```tsx
  const loggingEnabled = config?.control_execution_logging?.value === 'true';
  // ...
  {loggingEnabled && (
      <Button onClick={() => setShowLogModal(true)}>Log Execution</Button>
  )}
  ```
- Also hide the "Recent Executions" tab if disabled.

#### 3. Feature UI: KRI Modal
**File:** `frontend/src/components/kri/RecordValueModal.tsx`
**Action:**
- Fetch config.
- `const strictMode = config?.kri_strict_period_submission?.value === 'true';`
- If `!strictMode`:
    - Show a Date Picker field "Submission Date" (or "Period End").
    - Allow user to pick a date.
- If `strictMode`:
    - Hide Date Picker.
    - Show strict text: "Value will be recorded for period ending {calculatedDate}".
