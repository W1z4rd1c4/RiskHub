# Plan 70-07: Risk Hub Integration & Polish

## Goal
Address user-reported issues regarding the visibility of soft-deleted items (roles, departments) in selection dropdowns and ensure comprehensive activity logging for administrative actions. This phase aims to "connect everything" by ensuring that Risk Hub configurations (active/inactive status) are correctly respected throughout the User Management and Data Entry workflows.

## Proposed Changes

### Backend Changes

#### [MODIFY] [access.py](file:///Users/stefanlesnak/Antigravity/Risk App/backend/app/api/v1/endpoints/access.py)
- Update `GET /access/roles`:
    - Filter returned roles to only include `is_active=True`.
    - This ensures deleted roles do not appear in the User Edit modal.
- Verification checks for `GET /access/users`:
    - Ensure filtered lists respect active status unless specifically requested (though users usually need to see all users).
