# Plan 70-11: Control Execution Logging Toggle

## Goal
Implement a global feature flag `control_execution_logging`. When disabled, the endpoints for recording control executions should be completely blocked to prevent usage, allowing the organization to hide this complexity if they don't use the module.

## Technical Guidance
This is a standard Feature Flag implementation.
- **Enabled**: Standard behavior.
- **Disabled**: Endpoint returns `403 Forbidden` or `404 Not Found` (effectively hiding the feature).

## Proposed Changes

### Backend

#### 1. Gate the Endpoint
**File:** `backend/app/api/v1/endpoints/controls.py`
**Function:** `log_execution` (POST `/{control_id}/executions`)
**Logic:**
```python
logging_enabled = await get_config_bool(db, "control_execution_logging", default=True)
if not logging_enabled:
    raise HTTPException(
        status_code=400, 
        detail="Control execution logging is disabled by system configuration."
    )
```

**Function:** `list_executions` (GET `/{control_id}/executions`)
**Logic:**
- Optionally gate this too, or return an empty list with a warning header. Gating is cleaner.
