# Plan 70-09: Enhanced Permission Scopes (Auditor Access)

## Goal
Implement a granular `departments:read_all` permission. This allows specialized roles (e.g., Auditors, Regulators, Senior Management) to view data across *all* departments without granting them full System Administrator privileges (which include write access and user management).

## Technical Guidance
Currently, the system assumes `Department Visibility == User's Assigned Department`. Exceptions are hardcoded to `is_privileged_user()` (which checks `AccessScope.GLOBAL`).

**The Problem**: We cannot make a "ReadOnly Global Viewer" because `is_privileged_user()` is often conflated with Admin power.
**The Fix**: Decouple "Global View" from "Admin Scope". We will support a specific permission `departments:read_all` that overrides the department filter.

## Proposed Changes

### Backend

#### 1. Define New Permission
**File:** `backend/app/core/security.py` or `backend/app/db/init_db.py` (Seeding)
**Action**:
- Add `departments:read_all` to the list of available system permissions.
- Create a new System Role: **"Internal Auditor"**.
    - Permissions: `risks:read`, `controls:read`, `kris:read`, `departments:read_all`. (Note: Read-only).

#### 2. Update Scope Logic
**File:** `backend/app/core/permissions.py`
**Action:** Modify `get_user_department_ids(user: User) -> Optional[list[int]]`.
**Logic:**
```python
def get_user_department_ids(user: User) -> Optional[list[int]]:
    # 1. Check strict privilege (existing)
    if is_privileged_user(user):
        return None  # All depts
    
    # 2. NEW: Check explicit 'read_all' permission
    if has_permission(user, "departments", "read_all"):
        return None  # All depts
        
    # 3. Fallback to assigned department (existing logic)
    # ...
```

#### 3. Audit Endpoint Dependencies
**Files:** `backend/app/api/v1/endpoints/departments.py`, `backend/app/api/v1/endpoints/risks.py`
**Action**:
- Review `list_departments`. Ensure it uses `get_user_department_ids` for filtering (it does).
- **Crucial**: Ensure it doesn't have an early return like `if not current_user.is_superuser: filter()`. It must strictly rely on the helper function modified above.

## Verification Plan

### Automated Tests (Pytest)
**File:** `backend/tests/core/test_permission_scope_enhanced.py` (Create new)

1.  **`test_scope_regular_user`**:
    - Create User in Dept A.
    - Assert `get_user_department_ids` returns `[DeptA.id]`.

2.  **`test_scope_auditor_user`**:
    - Create User in Dept A.
    - Assign permission `departments:read_all`.
    - Assert `get_user_department_ids` returns `None` (Global).

3.  **`test_endpoint_auditor_visibility`**:
    - Create Dept A and Dept B.
    - User assigned to Dept A but has `departments:read_all`.
    - Call `GET /api/v1/departments`.
    - **Assert**: Response includes Dept A *and* Dept B.
