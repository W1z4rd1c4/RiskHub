# Plan: i18n Infrastructure Setup

## Objective
Install and configure the internationalization (i18n) infrastructure for the React frontend, including the translation library, language context provider, and locale switching mechanism.

## Execution Context
- **Phase**: 20 Czech Localization
- **Plan**: 20-01
- **Depends on**: Phase 15 (Settings Page with LocalizationSettings placeholder)

## Tasks

### 1. Install i18n Library
- [ ] Install `react-i18next` and `i18next` packages
- [ ] Install `i18next-browser-languagedetector` for automatic language detection
- [ ] Add TypeScript types if needed

### 2. Create i18n Configuration
- [ ] Create `frontend/src/i18n/index.ts` with i18next initialization
- [ ] Configure language detection (localStorage preference first, then browser language)
- [ ] Configure fallback language (English)
- [ ] Enable debug mode for development

### 3. Create Translation File Structure
- [ ] Create `frontend/src/i18n/locales/` directory
- [ ] Create `frontend/src/i18n/locales/en/` directory for English translations
- [ ] Create `frontend/src/i18n/locales/cs/` directory for Czech translations
- [ ] Create namespace structure:
  - `common.json` - Shared UI elements (buttons, labels, errors)
  - `navigation.json` - Sidebar and navigation items
  - `dashboard.json` - Dashboard page content
  - `risks.json` - Risk management content
  - `controls.json` - Control management content
  - `kris.json` - KRI management content
  - `approvals.json` - Approval workflow content
  - `settings.json` - Settings page content
  - `admin.json` - Admin console content
  - `auth.json` - Authentication content

### 4. Create TypeScript Types
- [ ] Create `frontend/src/i18n/types.ts` with translation key types
- [ ] Export type-safe translation hook wrapper

### 5. Integrate with App
- [ ] Import and initialize i18n in `main.tsx`
- [ ] Wrap App component with I18nextProvider
- [ ] Update LocalizationSettings to use i18n language switching

### 6. Create Translation Helper Hooks
- [ ] Create `useTranslation` wrapper with namespace support
- [ ] Create `useFormattedDate` hook for locale-aware date formatting
- [ ] Create `useFormattedNumber` hook for locale-aware number formatting

## Verification
- [ ] Run `npm run build` to verify no TypeScript errors
- [ ] Verify language switching works in Settings → Localization
- [ ] Verify localStorage preference persists across sessions
- [ ] Verify fallback to English when translation key missing

## Success Criteria
- [ ] i18next library installed and configured
- [ ] Translation file structure created with English and Czech directories
- [ ] LocalizationSettings updates i18n language dynamically
- [ ] Type-safe translation hooks available

## Output
- `frontend/src/i18n/index.ts`
- `frontend/src/i18n/types.ts`
- `frontend/src/i18n/locales/en/*.json` (10 namespace files, English strings)
- `frontend/src/i18n/locales/cs/*.json` (10 namespace files, initially same as English)
- Updated `frontend/src/main.tsx`
- Updated `frontend/src/components/settings/LocalizationSettings.tsx`
