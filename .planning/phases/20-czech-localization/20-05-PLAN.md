# Plan: Backend API Messages Translation

## Objective
Internationalize backend API error messages, validation messages, and response texts. Add language header support and configurable locale for API responses.

## Execution Context
- **Phase**: 20 Czech Localization
- **Plan**: 20-05
- **Depends on**: Plan 20-04 (Admin Pages Translation)

## Tasks

### 1. Backend i18n Infrastructure
- [ ] Research Python i18n approach (gettext vs custom dictionary)
- [ ] Create `backend/app/i18n/` directory
- [ ] Create `backend/app/i18n/__init__.py` with translation loader
- [ ] Create translation dictionaries:
  - `backend/app/i18n/en.py` - English messages
  - `backend/app/i18n/cs.py` - Czech messages

### 2. Language Detection Middleware
- [ ] Create language middleware in `backend/app/middleware/language.py`
- [ ] Detect language from:
  1. `Accept-Language` header
  2. User preference stored in database (if authenticated)
  3. Default to English
- [ ] Add language to request state for endpoint access

### 3. Translate Validation Messages
- [ ] Pydantic validation error messages customization
- [ ] Create centralized validation message dictionary
- [ ] Key messages:
  - "Field is required" → "Pole je povinné"
  - "Must be a valid email" → "Musí být platný email"
  - "Value must be between X and Y" → "Hodnota musí být mezi X a Y"
  - "Invalid format" → "Neplatný formát"

### 4. Translate API Error Messages
- [ ] Create `backend/app/i18n/errors.py` with error message dictionary
- [ ] Update exception handlers to use translated messages
- [ ] Key error messages:
  - "Not found" → "Nenalezeno"
  - "Access denied" → "Přístup odepřen"
  - "Unauthorized" → "Neautorizováno"
  - "Invalid credentials" → "Neplatné přihlašovací údaje"
  - "Session expired" → "Relace vypršela"
  - "Rate limit exceeded" → "Překročen limit požadavků"

### 5. Translate Business Logic Messages
- [ ] Approval workflow messages:
  - "Approval request created" → "Žádost o schválení vytvořena"
  - "Request approved" → "Žádost schválena"
  - "Request rejected" → "Žádost zamítnuta"
  - "Cannot approve own request" → "Nelze schválit vlastní žádost"
- [ ] Notification messages:
  - "KRI value due" → "KRI hodnota k odeslání"
  - "Approval required" → "Vyžadováno schválení"
  - "Breach detected" → "Zjištěno překročení"

### 6. Translate Activity Log Descriptions
- [ ] Update activity log generation to use translated descriptions
- [ ] Action descriptions:
  - "Created risk" → "Vytvořeno riziko"
  - "Updated control" → "Aktualizována kontrola"
  - "Archived KRI" → "Archivován KRI"
  - "Approved request" → "Schválena žádost"

### 7. Update API Response Helpers
- [ ] Create `backend/app/core/i18n.py` with translation helpers
- [ ] Update response builders to accept locale parameter
- [ ] Ensure all API messages pass through translation layer

## Verification

- [ ] Verify error messages return in Czech
- [ ] Verify validation messages return in Czech
- [ ] Test activity log descriptions in Czech

- [ ] All error messages translatable
