# Plan 03.2-01: List View Enhancements

## Objective
Enhance the Risk Register and Controls pages with grouped views, improved pagination, and column sorting for better data exploration.

## Scope

### In Scope
- Grouped view modes (by Category, Department, Process)
- View switcher tabs in page header
- Pagination showing "Page X of Y" with total count
- Clickable column headers for ascending/descending sort
- Apply to both Risks and Controls pages

### Out of Scope
- Server-side sorting (client-side for MVP)
- Keyboard shortcuts
- Column resizing/reordering

## Tasks

### Task 1: Create SortableTable Component
Create reusable table component with:
- Clickable column headers
- Sort indicator icons (▲/▼)
- Toggle between asc/desc on click
- Generic typing for any data type

**File:** `frontend/src/components/tables/SortableTable.tsx`

### Task 2: Create GroupedView Component
Create component that groups data by field:
- Collapsible group sections
- Group header with item count
- Supports grouping by: category, department, process

**File:** `frontend/src/components/tables/GroupedView.tsx`

### Task 3: Create ViewSwitcher Component
Create tab-style view selector:
- Options: "All", "By Category", "By Department", "By Process"
- Persists selection in URL params or local state
- Styled consistently with existing filter bar

**File:** `frontend/src/components/tables/ViewSwitcher.tsx`

### Task 4: Create Pagination Component
Create pagination with page info:
- Shows "Page 1 of 5"
- Total item count
- Prev/Next buttons (existing style)
- Optional: page number buttons

**File:** `frontend/src/components/tables/Pagination.tsx`

### Task 5: Update RisksPage
Integrate new components:
- Add ViewSwitcher above table
- Replace table with SortableTable
- Add GroupedView for grouped modes
- Update pagination component

**File:** `frontend/src/pages/RisksPage.tsx`

### Task 6: Update ControlsPage
Integrate same components:
- Add ViewSwitcher
- Replace table with SortableTable  
- Add GroupedView
- Update pagination

**File:** `frontend/src/pages/ControlsPage.tsx`

### Task 7: Backend Total Count (if needed)
If API doesn't return total count:
- Add `total` field to list responses
- Or add count endpoint

**Files:** `backend/app/api/v1/endpoints/risks.py`, `controls.py`

- [ ] Column headers are clickable and sort data
- [ ] Sort direction toggles on repeated click
- [ ] View switcher shows 4 modes (All, Category, Department, Process)
- [ ] Grouped views collapse/expand
- [ ] Pagination shows current page and total pages
- [ ] Works on both Risks and Controls pages
