# Plan: Database Schema & Migration (Risk Name)

## Objective
Add a mandatory `name` field to the Risk model and database, backfilling existing records.

## Execution Context
- **Phase**: 200 Entity Naming Enforcement
- **Plan**: 200-01

## Tasks

### 1. Backend Model Update
- [ ] Modify `backend/app/models/risk.py`:
    - Add `name: Mapped[str] = mapped_column(String(255), index=True, nullable=True)` (temporary nullable for migration).
- [ ] Modify `backend/app/models/key_risk_indicator.py`:
    - Add `description: Mapped[str] = mapped_column(Text, nullable=True)`.

### 2. Database Migration (Alembic)
- [ ] Generate migration script: `alembic revision --autogenerate -m "Add name to risks and description to kris"`
- [ ] Edit migration script:
    - Risk Name:
        - Add `op.add_column('risks', sa.Column('name', sa.String(length=255), nullable=True))`
        - Update existing risks: `UPDATE risks SET name = process` (or similar fallback).
        - Alter to non-nullable: `op.alter_column('risks', 'name', nullable=False)`
    - KRI Description:
        - Add `op.add_column('key_risk_indicators', sa.Column('description', sa.Text(), nullable=True))`
        - Update existing KRIs: `UPDATE key_risk_indicators SET description = metric_name` (or empty string/ "No description").
        - Alter to non-nullable (if we decide to enforce it). Let's enforce it: `op.alter_column('key_risk_indicators', 'description', nullable=False)`.

### 3. Verification
- [ ] Run migration `alembic upgrade head`.
- [ ] Inspect database to ensure all risks have names and KRIs have descriptions.

## Output
- Updated `Risk` model.
- Applied migration.
