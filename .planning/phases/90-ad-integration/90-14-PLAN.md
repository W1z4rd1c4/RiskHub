# Phase 90-14: Uncategorised Items Governance

## Objective

Ensure that Risks, Controls, and KRIs assigned to the "Uncategorised" department are visible in the Governance "Orphaned Items" list and can be resolved (reassigned).

## Execution Context

@backend/app/services/orphaned_item_service.py
@backend/app/api/v1/endpoints/orphaned_items.py

## Context

### Problem
- Items moved to "Uncategorised" department (Phase 90-13) are technically "orphaned" but might not have a record in the `orphaned_items` table if they weren't flagged by user deactivation.
- These items are currently invisible to the Governance workflow.
- User specifically mentioned "uncategorised risks, controls or KRI".

### Solution
- **Strategy**: Auto-register Uncategorised items as Orphans.
- Modify `OrphanedItemService` to perform a "Sweep" for Uncategorised items.
- If a Risk/Control is in 'UNCAT' department and has no `orphaned_items` entry -> Create one.
- **KRI Handling**:
    - KRIs are typically children of Risks.
    - If the parent Risk is Uncategorised, it should be flagged.
    - Need to check if KRIs can exist independently or have their own department/owner. (Assuming they follow Risk).
    - If KRIs can be orphaned independently, we need to handle `item_type="kri"`.

## Tasks

### Task 1: KRI Support (Backend)
- [ ] Check if `orphaned_items` supports `item_type="kri"`.
- [ ] If KRIs have independent owners/departments, add logic to flag them.
- [ ] Update `OrphanedItemService` to handle KRI resolution (reassigning).

### Task 2: Uncategorised Sweep Logic
- [ ] Implement `scan_uncategorised_items(db)` in `OrphanedItemService`.
    - Find all Risks/Controls where `department.code == 'UNCAT'`.
    - Filter out IDs already in `orphaned_items`.
    - Bulk insert new `OrphanedItem` records.
- [ ] Call this sweep method in:
    - `get_pending_orphans` (Lazy sync? Or background job?) -> Better to call it on `list_orphaned_items` API hit to ensure freshness.

### Task 3: Trigger Sweep
- [ ] Update `GET /api/v1/orphaned-items/` to await `scan_uncategorised_items()` before returning results.
