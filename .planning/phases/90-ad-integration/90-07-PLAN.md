<task type="auto">
  <name>Task 1: Create orphaned items API endpoints</name>
  <files>backend/app/api/v1/endpoints/orphaned_items.py, backend/app/api/v1/router.py</files>
  <action>
    Create new router `orphaned_items.py`:
    
    1. `GET /api/v1/orphaned-items/` - List pending orphaned items
       - Query params: item_type (optional filter), status (default "pending")
       - Include item details (risk/control name, previous owner info)
       - Paginated response
       - Requires admin or CRO role
    
    2. `GET /api/v1/orphaned-items/stats` - Get orphan statistics
       - Returns counts by item_type and status
       - Used for dashboard badge/widgets
       - Requires authenticated user
    
    3. `POST /api/v1/orphaned-items/{id}/resolve` - Resolve an orphan
       - Body: OrphanedItemResolve (new_owner_id)
       - Updates the underlying item's owner
       - Marks orphan as resolved
       - Requires admin role
    
    4. `GET /api/v1/orphaned-items/{id}` - Get single orphan detail
       - Include full item details
       - Requires admin or CRO role
    
    Register router in router.py.
  </action>
  <verify>
    1. Start backend
    2. curl http://localhost:8000/api/v1/orphaned-items/ -H "Authorization: Bearer <admin_token>"
    3. Check endpoint returns list (empty or with data)
    4. Check /docs shows new endpoints
  </verify>
  <done>All four orphaned item endpoints working with auth.</done>
</task>
