# Phase 90-11: Uncategorised Department Fallback

## Objective

Implement an "Uncategorised" department fallback so that when an orphaned risk/control's department becomes empty (the last user was removed), the item is automatically reassigned to a default "Uncategorised" department.

## Execution Context

@backend/app/models/department.py
@backend/app/models/risk.py
@backend/app/models/control.py
@backend/app/services/orphaned_item_service.py
@backend/app/api/v1/endpoints/orphaned_items.py

## Context

### Problem Statement
When a user who owns a risk/control is deactivated AND they were the only employee in their department, the orphan resolution UI shows no valid department to assign the item to. The risk becomes "homeless" with no clear organizational owner.

### Solution
Create a system-reserved "Uncategorised" department that:
1. Is seeded automatically on database initialization
2. Cannot be deleted by users
3. Serves as fallback when resolving orphans with missing departments
4. Is displayed with a warning indicator in the UI

## Tasks

### Task 1: Seed Uncategorised Department
- [ ] Add migration or seed script to ensure "Uncategorised" department exists (code: `UNCAT`, id: reserved low number like 1)
- [ ] Mark department as `is_system = True` (add column if needed) to prevent deletion

### Task 2: Update Orphan Resolution Logic
- [ ] In `OrphanedItemService.resolve_orphan()`, if the new owner's department is null, assign item to "Uncategorised"
- [ ] Add optional `department_id` parameter to resolve endpoint to allow explicit department override

### Task 3: Update Resolve Modal UI
- [ ] Add department dropdown alongside user selection in ResolveOrphanModal
- [ ] Default to new owner's department, with fallback to "Uncategorised"
- [ ] Show warning badge when "Uncategorised" is selected

### Task 4: Governance Page Enhancement
- [ ] Add "Uncategorised Items" count to Governance stats
- [ ] Filter/highlight items in "Uncategorised" department
