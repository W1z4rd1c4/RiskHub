Create orphan flagging system to track risks, controls, and KRIs that have lost their owners.

Purpose: When a user is deactivated and they owned risks/controls/KRIs, those items need to be flagged for administrative attention. This phase adds the data model and flagging logic.

Output: OrphanedItem model/table, automatic flagging service, and API endpoints to query orphaned items.

    ```python
    class OrphanedItem(Base):
        __tablename__ = "orphaned_items"
        
        id: Mapped[int] = mapped_column(primary_key=True)
        
        # What type of item lost its owner
        item_type: Mapped[str]  # "risk" | "control" | "kri"
        item_id: Mapped[int]    # FK to risks.id, controls.id, or key_risk_indicators.id
        
        # Who was the previous owner
        previous_owner_id: Mapped[int] = mapped_column(ForeignKey("users.id"))
        previous_owner: Mapped["User"] = relationship("User")
        
        # When did it become orphaned
        orphaned_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())
        
        # Resolution
        resolved_at: Mapped[datetime | None] = mapped_column(DateTime, nullable=True)
        resolved_by_id: Mapped[int | None] = mapped_column(ForeignKey("users.id"), nullable=True)
        new_owner_id: Mapped[int | None] = mapped_column(ForeignKey("users.id"), nullable=True)
        
        # Status
        status: Mapped[str] = mapped_column(String(20), default="pending")  # "pending" | "resolved"
    ```

<task type="auto">
  <name>Task 2: Create OrphanedItemService for flagging and resolution</name>
  <files>backend/app/services/orphaned_item_service.py, backend/app/schemas/orphaned_item.py</files>
  <action>
    Create `OrphanedItemService`:
    
    1. `flag_orphaned_items(db, user_id: int) -> list[OrphanedItem]`:
       - Query risks where owner_id = user_id
       - Query controls where control_owner_id = user_id
       - Query KRIs via their risk's owner (optional - KRIs don't have direct owner)
       - Create OrphanedItem records for each
       - Return created records
    
    2. `get_pending_orphans(db, item_type: str | None) -> list[OrphanedItem]`:
       - Query all orphaned_items where status="pending"
       - Optionally filter by item_type
       - Include relationships to get item details
    
    3. `resolve_orphan(db, orphan_id: int, new_owner_id: int, resolved_by_id: int) -> OrphanedItem`:
       - Update the actual item (risk/control) with new owner_id
       - Mark orphaned_item as resolved
       - Record who resolved it and new owner
    
    Create Pydantic schemas:
    - OrphanedItemRead (for listing)
    - OrphanedItemResolve (for resolution request: new_owner_id)
    - OrphanedItemStats (counts by type and status)
  </action>
  <verify>Import service in Python REPL and verify no syntax errors.</verify>
  <done>OrphanedItemService with flag, get, and resolve methods. Schemas defined.</done>
