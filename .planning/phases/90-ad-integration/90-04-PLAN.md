<objective>
Implement webhook infrastructure in AD Emulator to push change events to RiskHub.

Purpose: Enable real-time synchronization from AD Emulator to RiskHub by sending webhook notifications when directory users are created, updated, or deactivated. Currently, RiskHub polls AD Emulator manually; this plan establishes the push mechanism.

Output: AD Emulator backend with webhook dispatch capability, RiskHub webhook receiver endpoint, and configuration for webhook target URL.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Add webhook configuration to AD Emulator backend</name>
  <files>AD Emulator/backend/app/config.py, AD Emulator/backend/app/.env.example</files>
  <action>
    Add `webhook_target_url` setting to AD Emulator's config (optional, nullable). When set, AD Emulator will POST change events to this URL.
    
    Add a new `WebhookPayload` Pydantic schema in `AD Emulator/backend/app/schemas/webhook.py`:
    ```
    - event_type: Literal["user.created", "user.updated", "user.deactivated", "user.activated"]
    - timestamp: datetime (UTC)
    - data: DirectoryUserRead (the affected user)
    ```
    
    Create `AD Emulator/backend/app/services/webhook_service.py`:
    - `dispatch_event(event_type, user_data)` async function
    - Uses httpx.AsyncClient to POST to webhook_target_url
    - 5-second timeout, fire-and-forget (no blocking on failures)
    - Log success/failure but don't raise exceptions (AD operations should succeed regardless of webhook)
  </action>
  <verify>Run AD Emulator backend - no startup errors. Check `/docs` shows no changes to existing endpoints.</verify>
  <done>WebhookPayload schema defined, webhook_service.py created with dispatch_event function, config.py has webhook_target_url setting.</done>
</task>

<task type="auto">
  <name>Task 2: Dispatch webhooks on CRUD operations in AD Emulator</name>
  <files>AD Emulator/backend/app/api/endpoints/users.py</files>
  <action>
    Modify the existing user CRUD endpoints to call webhook dispatch after successful database operations:
    
    1. `create_directory_user`: After `db.commit()`, call `await dispatch_event("user.created", new_user)` 
    2. `update_directory_user`: After `db.commit()`, call `await dispatch_event("user.updated", user)`
    3. `deactivate_directory_user`: After setting `account_enabled=False` and commit, call `await dispatch_event("user.deactivated", user)`
    4. `activate_directory_user`: After setting `account_enabled=True` and commit, call `await dispatch_event("user.activated", user)`
    
    Import dispatch_event from webhook_service. Wrap dispatch calls in try/except to ensure AD operations complete even if webhook fails.
    
    Important: Only dispatch if `settings.webhook_target_url` is configured (not None/empty).
  </action>
  <verify>
    1. Start AD Emulator with `WEBHOOK_TARGET_URL=http://localhost:8000/api/v1/directory/webhook` 
    2. Create a user via AD Emulator UI or curl
    3. Check AD Emulator logs for webhook dispatch attempt
  </verify>
  <done>All four user endpoints (create, update, deactivate, activate) dispatch webhook events when webhook_target_url is configured.</done>
</task>

<task type="auto">
  <name>Task 3: Create webhook receiver endpoint in RiskHub</name>
  <files>backend/app/api/v1/endpoints/directory.py, backend/app/schemas/directory_sync.py</files>
  <action>
    Add a new endpoint `POST /api/v1/directory/webhook` in RiskHub:
    
    1. Create `WebhookPayload` schema in `directory_sync.py` (matching AD Emulator's schema)
    2. Create `@router.post("/webhook")` endpoint:
       - Accept WebhookPayload body
       - For now, just log the event type and user external_id
       - Return 200 OK immediately (acknowledge receipt)
       - No authentication for now (will add in later phase)

    3. Check response is 200 and backend logs show received event
  </verify>
  <done>RiskHub has POST /api/v1/directory/webhook endpoint that accepts and logs webhook events.</done>
