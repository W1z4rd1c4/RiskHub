# Phase 90-13: Empty Department Cleanup

## Objective

Hide departments that have 0 active users and move their associated risks/controls to the "Uncategorised" department. This ensures that "zombie" departments (like historical ones from inactive users) don't clutter the UI, while preserving their data in the database for integrity.

## Execution Context

@backend/app/services/directory_sync_service.py
@backend/app/api/v1/endpoints/departments.py
@backend/app/crud/crud_department.py

## Context

### Problem Statement
- Departments persist even when they have no active users (e.g., "MOMS" department with 1 inactive user).
- These empty departments clutter the UI.
- Risks in these departments might be overlooked.

### User Requirements
1.  **Trigger**: If a department has 0 active users.
2.  **Action 1 (Data)**: Move all Risks and Controls from this department to the "Uncategorised" department.
3.  **Action 2 (UI)**: The department should be "gone" (hidden) from the UI.
4.  **Constraint**: Do NOT delete the department record from the database (data integrity).

## Tasks

### Task 1: Implement Cleanup Logic
- [ ] Add `cleanup_empty_departments(db: AsyncSession)` to `DirectorySyncService`.
    - Find all departments where `active_user_count == 0` AND `is_system == False`.
    - Function to bulk update `Risk.department_id` -> `UNCAT.id` for these departments.
    - Function to bulk update `Control.department_id` -> `UNCAT.id` for these departments.
- [ ] Invoke this cleanup after `run_sync` (full sync) and `sync_single_user` (incremental sync).

### Task 2: API Filtering
- [ ] Update `get_departments` (list) endpoint to filter out "zombie" departments.
    - Filter logic: Include department IF `active_user_count > 0` OR `is_system == True`.
    - Ensure newly created empty departments (if any manual creation exists) aren't hidden immediately? (Assuming sync-only for now based on `description="Imported from directory sync"`).
