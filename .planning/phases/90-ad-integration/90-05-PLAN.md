<objective>
Implement automatic synchronization when RiskHub receives webhook events from AD Emulator.

Purpose: When AD Emulator sends a webhook notification about a user change, RiskHub should automatically process the change (create/update/deactivate the corresponding RiskHub user) without manual intervention.

Output: RiskHub webhook endpoint that triggers automatic sync for the affected user, with orphan detection for deactivated users.
</objective>

<task type="auto">
  <name>Task 1: Implement single-user sync function in DirectorySyncService</name>
  <files>backend/app/services/directory_sync_service.py</files>
  <action>
    Add a new method `sync_single_user(db: AsyncSession, user_data: dict, event_type: str)` to DirectorySyncService:
    
    1. Extract logic from _run_sync to handle a single user
    2. For "user.created" or "user.updated": 
       - Find existing user by external_id
       - If exists: update fields (email, display_name, department, etc.)
       - If not exists: create new user (similar to current sync logic)
    3. For "user.deactivated":
       - Find user by external_id
       - Set is_active=False
       - Return list of affected items (risks, controls, KRIs) that lost their owner
    4. For "user.activated":
       - Find user by external_id
       - Set is_active=True
    
    Return a dict with: {"action": "created"|"updated"|"deactivated"|"activated", "user_id": int, "orphaned_items": {...}}
    
    Important: Reuse existing helper functions (_normalize_email, _get_or_create_department, etc.)
  </action>
  <verify>Unit test or manual verification that sync_single_user handles each event type correctly.</verify>
  <done>DirectorySyncService.sync_single_user method implemented handling all four event types.</done>
</task>

<task type="auto">
  <name>Task 2: Enhance webhook endpoint to trigger automatic sync</name>
  <files>backend/app/api/v1/endpoints/directory.py, backend/app/schemas/directory_sync.py</files>
  <action>
    Modify the webhook endpoint from phase 90-04:
    
    1. Inject database session dependency
    2. Instantiate DirectorySyncService

    5. Return response with sync result: {"status": "processed", "action": "created"|"updated"|..., "orphaned_count": int}
