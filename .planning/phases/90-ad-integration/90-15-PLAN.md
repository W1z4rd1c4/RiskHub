# Phase 90-15: Governance UI Redesign & KRI Orphans

## Objective

Redesign the Governance Page to match the user's specific "4-bar" layout requirement and introduce logic to handle "Orphaned KRIs".

## Design Source
Reference mockup was kept outside the public repository; implement from the layout requirements below.

## Requirements
1.  **Navigation Bar Split**:
    - **Bar 1: Risks**: "Pending Orphans" (Risks with no department or no owner). Default View. Clikable.
    - **Bar 2: Controls**: "Orphaned Controls" (No department, no owner, or no risk linkage). Clickable.
    - **Bar 3: KRIs**: "Orphaned KRIs" (KRI not linked to a risk). Clickable.
    - **Bar 4: Total**: "Uncategorised" (Sum of all). Non-clickable summary.
2.  **Resolution UI**:
    - distinct resolution modal/action for each type.
    - KRI Resolution: Likely involves linking to a Risk or assigning a department (if we relax strict FKs).
3.  **Backend Changes**:
    - Support `item_type="kri"` in `OrphanedItem` model.
    - Implement `scan_broken_kris` logic (check for data integrity issues).
