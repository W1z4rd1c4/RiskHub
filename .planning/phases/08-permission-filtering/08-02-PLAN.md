Create `backend/app/api/v1/endpoints/approvals.py`:

**Endpoints to implement:**

1. `POST /api/v1/approvals` - Create new approval request
   - Any authenticated user with delete permission for resource type
   - Looks up resource (risk/control/kri), snapshots name
   - **Requires mandatory `reason` field** (not optional)
   - Creates ApprovalRequest with status=pending
   - Returns 201 with request details

2. `GET /api/v1/approvals` - List approval requests (Workflow Requests)
   - Risk Managers/CRO/Admin: see all pending
   - Other users: see only their own requests
   - Filter by: status, resource_type
   - Paginated response

3. `GET /api/v1/approvals/{id}` - Get single request
   - Permission: requester or Risk Manager

4. `POST /api/v1/approvals/{id}/approve` - Approve and execute deletion
   - Only Risk Manager, CRO, or Admin roles
   - **Requires mandatory `resolution_notes` field**
   - Sets status=approved, resolved_by, resolved_at
   - **AUTO-EXECUTES: Archives/deletes the resource immediately**
   - Returns 200 with updated request

5. `POST /api/v1/approvals/{id}/reject` - Reject request
   - Only Risk Manager, CRO, or Admin roles
   - **Requires mandatory `resolution_notes` field**
   - Sets status=rejected
   - Resource returns to normal state (no pending flag)
   - Returns 200 with updated request

6. `POST /api/v1/approvals/{id}/cancel` - Cancel request
   - Only original requester
   - Sets status=cancelled
   - Resource returns to normal state

**Helper function:**
```python
def can_resolve_approvals(user: User) -> bool:
    """Check if user can approve/reject requests."""
    if not user.role:
        return False
    return user.role.name in {RoleType.RISK_MANAGER, RoleType.CRO, RoleType.ADMIN}
```

**Permission patterns:**
- Use `get_current_user` dependency for all endpoints
- Check `can_resolve_approvals()` for approve/reject
- Check ownership for cancel
- Validate resource exists before creating request

```python
import pytest
from httpx import AsyncClient

@pytest.mark.asyncio
async def test_create_approval_request(client: AsyncClient, test_risk):
    """Test creating approval request for risk deletion."""
    response = await client.post(
        "/api/v1/approvals",
        json={"resource_type": "risk", "resource_id": test_risk.id, "reason": "No longer applicable"}
    )
    assert response.status_code == 201
    data = response.json()
    assert data["status"] == "pending"
    assert data["resource_type"] == "risk"

@pytest.mark.asyncio
async def test_list_pending_approvals(client: AsyncClient):
    """Test listing pending approval requests."""
    response = await client.get("/api/v1/approvals?status=pending")
    assert response.status_code == 200
    data = response.json()
    assert "items" in data
    assert "total" in data

@pytest.mark.asyncio
async def test_approve_request(client_risk_manager: AsyncClient, pending_approval):
    """Test Risk Manager can approve request."""
    response = await client_risk_manager.post(
        f"/api/v1/approvals/{pending_approval.id}/approve",
        json={"resolution_notes": "Approved for deletion"}
    )
    assert response.status_code == 200
    assert response.json()["status"] == "approved"

@pytest.mark.asyncio
async def test_reject_request(client_risk_manager: AsyncClient, pending_approval):
    """Test Risk Manager can reject request."""
    response = await client_risk_manager.post(
        f"/api/v1/approvals/{pending_approval.id}/reject",
        json={"resolution_notes": "Still needed"}
    )
    assert response.status_code == 200
    assert response.json()["status"] == "rejected"

@pytest.mark.asyncio
async def test_non_manager_cannot_approve(client: AsyncClient, pending_approval):
    """Test regular user cannot approve."""
    response = await client.post(f"/api/v1/approvals/{pending_approval.id}/approve")
    assert response.status_code == 403
```
