Update to return proper HTTP status codes:

```python
from fastapi import HTTPException
from fastapi.responses import JSONResponse

@router.post("/webhook")
async def handle_webhook(...):
    try:
        await sync_user(...)
        return {"status": "ok", "message": "Sync completed"}
    except DatabaseConnectionError as e:
        # Transient failure - return 503 to trigger retry
        logger.error(f"Database unavailable during sync: {e}")
        raise HTTPException(status_code=503, detail="Sync temporarily unavailable")
    except ValidationError as e:
        # Permanent failure - return 400, no retry needed
        logger.warning(f"Invalid webhook payload: {e}")
        raise HTTPException(status_code=400, detail="Invalid payload")
    except Exception as e:
        # Unknown failure - return 500 to trigger retry
        logger.error(f"Sync failed: {e}")
        raise HTTPException(status_code=500, detail="Sync failed")
```

Error code semantics:

- 200: Success
- 400: Invalid payload (don't retry)
- 500: Unexpected error (retry)
- 503: Transient failure (retry with backoff)
