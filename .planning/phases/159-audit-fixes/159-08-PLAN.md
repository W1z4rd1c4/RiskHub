<task type="auto">
  <name>Task 1: Understand current CSP</name>
  <files>nginx.conf</files>
  <action>
Find the Content-Security-Policy header:

```nginx
add_header Content-Security-Policy "
    default-src 'self';
    connect-src 'self' http://backend:*;
    ...
" always;
```

Problems:

1. `http://backend:*` - Browsers can't resolve the Docker internal hostname
2. Wildcard port `:*` is overly permissive
3. If all API calls go through `/api/`, `'self'` is sufficient
  </action>

  <verify>Located the CSP header with http://backend:*.</verify>
  <done>Current CSP understood.</done>
</task>

<task type="auto">
  <name>Task 2: Simplify connect-src to 'self'</name>
  <files>nginx.conf</files>
  <action>
Update CSP:

```nginx
add_header Content-Security-Policy "
    default-src 'self';
    connect-src 'self';
    script-src 'self';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: blob:;
    font-src 'self';
    frame-ancestors 'none';
" always;
```

Rationale:

- All API calls go through `/api/` which is proxied to backend
- Same-origin requests need only `'self'`
- No external API calls required
  </action>
  <verify>CSP no longer includes http://backend:*.</verify>
  <done>CSP simplified.</done>
</task>
