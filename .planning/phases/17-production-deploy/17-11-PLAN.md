# Plan: Azure AD/Entra SSO Integration

## Objective
Implement Single Sign-On authentication using Azure AD/Entra ID, replacing mock JWT authentication for production.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-05
- **Depends on**: None (can be developed independently)

## Tasks

### 1. MSAL Backend Integration
- [ ] Add `msal` package to requirements.txt
- [ ] Create `backend/app/core/azure_auth.py`:
  - MSAL confidential client configuration
  - Token validation function
  - Azure AD configuration from environment
- [ ] Add Azure AD environment variables:
  - `AZURE_AD_TENANT_ID`
  - `AZURE_AD_CLIENT_ID`
  - `AZURE_AD_CLIENT_SECRET`

### 2. Authentication Endpoint Updates
- [ ] Update `backend/app/api/v1/endpoints/auth.py`:
  - Add `/auth/azure/login` endpoint (redirect to Azure)
  - Add `/auth/azure/callback` endpoint (handle token exchange)
  - Update `/auth/me` to validate Azure tokens
- [ ] Create session management for Azure-authenticated users

### 3. Frontend MSAL Integration
- [ ] Add `@azure/msal-browser` package
- [ ] Create `frontend/src/services/azureAuth.ts`:
  - MSAL configuration
  - Login/logout functions
  - Token acquisition for API calls
- [ ] Update `frontend/src/contexts/AuthContext.tsx`:
  - Azure login flow
  - Token refresh handling
  - Redirect handling

### 4. Environment-Based Auth Switching
- [ ] Add `AUTH_MODE` environment variable (mock/azure)
- [ ] Update backend to select auth provider based on config
- [ ] Update frontend to select auth provider based on config
- [ ] Keep mock auth working for development

### 5. User Provisioning on First Login
- [ ] On first Azure login, auto-create user in database
- [ ] Map Azure AD claims to RiskHub user fields:
  - email → email
  - preferred_username → name
  - oid → external_id (for sync)
- [ ] Assign default role for new users

## Verification
- [ ] Login redirects to Azure AD login page
- [ ] After Azure login, user lands in application
- [ ] User record created/updated in database
- [ ] API calls include valid Azure token
- [ ] Mock auth still works when AUTH_MODE=mock
