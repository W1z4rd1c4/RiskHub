# Plan: Production Hardening

## Objective
Implement security hardening measures including CORS, CSP, secrets management, and secure configuration.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-04
- **Depends on**: 19-01 to 19-03

## Tasks

### 1. CORS Configuration
- [ ] Update `backend/app/main.py` with environment-based CORS:
  - Development: Allow localhost origins
  - Production: Strict allowed origins from config
- [ ] Add CORS origin validation logging

### 2. Content Security Policy
- [ ] Add CSP headers to nginx configuration
- [ ] Add CSP headers to backend responses for API calls
- [ ] Configure report-uri for CSP violations (optional)

### 3. Security Headers
- [ ] Implement security headers middleware:
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `X-XSS-Protection: 1; mode=block`
  - `Strict-Transport-Security` (HSTS)
  - `Referrer-Policy: strict-origin-when-cross-origin`

### 4. Secrets Management
- [ ] Remove hardcoded JWT secret from config.py
- [ ] Update config to require JWT_SECRET from environment
- [ ] Add startup validation for required secrets
- [ ] Document Key Vault / environment variable setup

### 5. Rate Limiting
- [ ] Add rate limiting middleware for login endpoint
- [ ] Implement account lockout after failed attempts
- [ ] Add rate limiting to API endpoints (configurable)

### 6. Logging Hardening
- [ ] Ensure no sensitive data logged (passwords, tokens)
- [ ] Add request ID tracing to all log entries
- [ ] Configure log rotation for production
