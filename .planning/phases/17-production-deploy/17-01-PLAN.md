# Plan: Docker Scaffolding

## Objective
Create production-ready Docker configuration with multi-stage builds, Docker Compose, and health checks.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-01
- **Depends on**: None (foundational)

## Tasks

### 1. Backend Dockerfile
- [ ] Create `backend/Dockerfile` with multi-stage build:
  - Stage 1: Build dependencies with pip
  - Stage 2: Slim runtime image with only production deps
- [ ] Add non-root user for security
- [ ] Configure health check endpoint

### 2. Frontend Dockerfile
- [ ] Create `frontend/Dockerfile` with multi-stage build:
  - Stage 1: Node build with npm run build
  - Stage 2: nginx serving static files
- [ ] Create optimized nginx.conf for SPA routing
- [ ] Add gzip compression and cache headers

### 3. Docker Compose
- [ ] Create `docker-compose.yml` for local development:
  - Backend service (FastAPI)
  - Frontend service (nginx)
  - PostgreSQL service
  - Optional: AD Emulator service
- [ ] Create `docker-compose.prod.yml` with production overrides:
  - Environment variable configuration
  - Volume mounts for logs and data
  - Network configuration

### 4. Environment Configuration
- [ ] Create `.env.example` with all required variables documented
- [ ] Update backend to read all secrets from environment
- [ ] Add startup validation for required environment variables

### 5. Health Check Endpoint
- [ ] Add `/health` endpoint to backend returning:
  - Database connectivity status
  - Application version
  - Uptime

## Verification
- [ ] `docker-compose up` starts all services successfully
- [ ] Frontend accessible at localhost:80
- [ ] Backend API accessible at localhost:8000
- [ ] Health endpoint returns OK with database connected
