# Plan: VM Deployment Scripts

## Objective
Create deployment scripts for bare-metal or VM installations using systemd services and nginx reverse proxy.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-02
- **Depends on**: 19-01 (uses same environment patterns)

## Tasks

### 1. Systemd Service Files
- [ ] Create `deploy/systemd/riskhub-backend.service`:
  - Uvicorn with gunicorn workers
  - Environment file loading
  - Automatic restart on failure
  - Logging to journald
- [ ] Create `deploy/systemd/riskhub-scheduler.service`:
  - Background task scheduler (APScheduler)
  - Depends on backend service

### 2. Nginx Configuration
- [ ] Create `deploy/nginx/riskhub.conf`:
  - Frontend static file serving
  - API reverse proxy to backend
  - WebSocket support for future real-time features
  - SSL/TLS termination configuration
  - Security headers (CSP, X-Frame-Options, etc.)

### 3. Installation Script
- [ ] Create `deploy/install.sh`:
  - System prerequisites check (Python 3.11+, Node 18+, PostgreSQL)
  - Virtual environment setup
  - Dependencies installation
  - Database migration
  - Systemd service installation
  - Nginx configuration installation
- [ ] Make script idempotent (safe to re-run)

### 4. Configuration Management
- [ ] Create `deploy/config/riskhub.env.template`:
  - All environment variables with documentation
  - Sensible defaults where appropriate
- [ ] Create `deploy/config/README.md` explaining each setting

### 5. Backup & Maintenance Scripts
- [ ] Create `deploy/scripts/backup-db.sh` for PostgreSQL backup
- [ ] Create `deploy/scripts/update.sh` for zero-downtime updates

## Verification
- [ ] `install.sh` runs without errors on fresh Ubuntu 22.04
- [ ] Services start and pass health checks
- [ ] Frontend loads through nginx
- [ ] API requests proxied correctly
