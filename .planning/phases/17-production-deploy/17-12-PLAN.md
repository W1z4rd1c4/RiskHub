# Plan: AD User Directory Lookup

## Objective
Implement "Pick from AD" functionality allowing admins to search Azure AD directory and import users into RiskHub.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-06
- **Depends on**: 19-05 (Azure AD authentication)

## Tasks

### 1. Microsoft Graph API Integration
- [ ] Add `msgraph-sdk-python` or use direct REST calls
- [ ] Create `backend/app/services/graph_service.py`:
  - Search users by name/email
  - Get user details by ID
  - List users with pagination
- [ ] Configure Graph API permissions:
  - `User.Read.All` (for reading user directory)

### 2. Backend API Endpoints
- [ ] Create `backend/app/api/v1/endpoints/directory.py`:
  - `GET /directory/users/search?q={query}` - Search AD users
  - `GET /directory/users/{oid}` - Get AD user details
  - `POST /directory/users/{oid}/import` - Import AD user to RiskHub
- [ ] Add RBAC: Only Admin/CRO can access directory endpoints

### 3. Import User Flow
- [ ] When importing from AD:
  - Check if user already exists (by external_id/oid)
  - If exists, update info from AD
  - If new, create with mapped fields
- [ ] Map AD fields to RiskHub:
  - `id` (oid) → `external_id`
  - `displayName` → `name`
  - `mail` → `email`
  - `department` → lookup/create department
  - `jobTitle` → store for reference

### 4. Frontend User Picker
- [ ] Create `frontend/src/components/ADUserPicker.tsx`:
  - Search input with debounce
  - Results dropdown showing AD users
  - "Import" button for each user
- [ ] Integrate into UsersPage:
  - "Add from AD" button
  - Modal with ADUserPicker
  - Success notification on import

### 5. Environment Fallback
- [ ] When `AUTH_MODE=mock`, use AD Emulator endpoints
- [ ] When `AUTH_MODE=azure`, use Microsoft Graph

## Verification
- [ ] Search returns AD users matching query
- [ ] Import creates new user in database
- [ ] Re-import updates existing user
- [ ] AD Emulator mode still works for development

## Output
- `backend/app/services/graph_service.py`
- `backend/app/api/v1/endpoints/directory.py`
- `frontend/src/components/ADUserPicker.tsx`
- Updated UsersPage with AD import button
