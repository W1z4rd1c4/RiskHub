# Plan: Azure Deployment

## Objective
Create Azure deployment configuration using Infrastructure as Code for App Service or Container Apps.

## Execution Context
- **Phase**: 19 Production Deployment & Enterprise Integration
- **Plan**: 19-03
- **Depends on**: 19-01 (Docker images)

## Tasks

### 1. Azure Resource Templates
- [ ] Create `deploy/azure/main.bicep` with:
  - Resource Group
  - App Service Plan (Linux)
  - Web App for Backend (Container)
  - Static Web App for Frontend
  - Azure Database for PostgreSQL Flexible Server
  - Key Vault for secrets
  - Application Insights for monitoring

### 2. Container Registry Setup
- [ ] Create `deploy/azure/acr.bicep` for Azure Container Registry
- [ ] Create GitHub Actions workflow for building and pushing images:
  - `.github/workflows/build-and-push.yml`

### 3. Deployment Pipeline
- [ ] Create `.github/workflows/deploy-azure.yml`:
  - Build Docker images
  - Push to ACR
  - Deploy to App Service
  - Run database migrations
  - Health check verification

### 4. Environment Configuration
- [ ] Create `deploy/azure/parameters.prod.json` for production settings
- [ ] Create `deploy/azure/parameters.staging.json` for staging
- [ ] Document Key Vault secret configuration

### 5. Networking & Security
- [ ] Configure VNet integration for database access
- [ ] Set up managed identity for Key Vault access
- [ ] Configure custom domain and SSL certificate

## Verification
- [ ] `az deployment group create` provisions all resources
- [ ] Application accessible via Azure-provided domain
- [ ] Database connection verified through managed identity
- [ ] Application Insights receiving telemetry

## Output

- `deploy/azure/acr.bicep`
- `deploy/azure/parameters.prod.json`
