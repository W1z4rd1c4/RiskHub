<task type="auto">
  <name>Task 1: Create Dashboard Schemas</name>
  <files>backend/app/schemas/dashboard.py</files>
  <action>
Create Pydantic response schemas for dashboard endpoints:

1. **DashboardSummaryResponse** - Overview stats:
   - `total_controls: int`
   - `controls_by_status: dict[str, int]` (draft, active, inactive, archived)
   - `controls_by_form: dict[str, int]` (manual, automatic)
   - `controls_by_frequency: dict[str, int]` (daily, weekly, monthly, etc.)
   - `total_risks: int`
   - `risks_by_status: dict[str, int]` (active, monitoring, closed, archived)
   - `critical_risks_count: int` (net_score >= 15)
   - `average_net_risk_score: float`

2. **DepartmentMetrics** - Per-department stats:
   - `department_id: int`
   - `department_name: str`
   - `control_count: int`
   - `risk_count: int`
   - `high_risk_count: int` (risk_level >= 4)
   - `compliance_rate: float` (active controls / total controls)

3. **RiskDistributionResponse** - For heatmap/matrix:
   - `distribution: list[dict]` with `probability`, `impact`, `count`

4. **ControlFrequencyTrend** - Time series for charts:
   - `period: str` (e.g., "2025-W24")
   - `execution_count: int`

Use Pydantic v2 syntax (model_config not Config class). Import from `pydantic` directly.

<task type="auto">
  <name>Task 2: Implement Dashboard Endpoints</name>
  <files>backend/app/api/v1/endpoints/dashboard.py, backend/app/api/v1/router.py</files>
  <action>
Create `dashboard.py` router with endpoints:

1. **GET /dashboard/summary** → `DashboardSummaryResponse`
   - Aggregate counts from controls and risks tables
   - Use SQLAlchemy `func.count()` with appropriate grouping
   - Critical risks = WHERE net_score >= 15
   - Calculate average net risk score

2. **GET /dashboard/departments** → `list[DepartmentMetrics]`
   - Group controls and risks by department
   - Join with departments table for names
   - Calculate compliance_rate as (active_controls / total_controls)

3. **GET /dashboard/risk-distribution** → `RiskDistributionResponse`
   - Group risks by (net_probability, net_impact) combinations
   - Count risks in each cell for 5x5 risk matrix visualization

4. **GET /dashboard/control-trends** → `list[ControlFrequencyTrend]`
   - Query control_executions grouped by ISO week
   - Last 8 weeks of data
   - Use SQLAlchemy `func.date_trunc('week', ControlExecution.executed_at)`

Add RBAC dependency injection - all dashboard endpoints require at minimum 'viewer' role. Use existing `get_current_user` pattern from other endpoints.

Update `router.py` to include: `api_router.include_router(dashboard.router, prefix="/dashboard", tags=["dashboard"])`
