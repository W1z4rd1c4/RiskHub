  <files>.planning/phases/90-ad-emulator/90-DISCOVERY.md</files>
  <action>
    Create a concise mapping spec that will drive all sync logic. Include:
    - Current RiskHub user fields and constraints (from models/schemas): email, name, is_active, role_id, department_id, manager_id.
    - Directory attributes to support: external_id, user_principal_name (UPN), email, display_name, given_name, surname, department, job_title, manager_external_id, account_enabled, source_payload.
    - Matching order: 1) email (lowercased), 2) UPN; when both missing → skip.
    - Conflict rules: directory is source of truth for name/email/is_active/department/manager; NEVER overwrite role_id unless a role mapping is explicitly configured.
    - Department resolution: match by name (case-insensitive), create if missing; store department_id on user.
    - Manager resolution: match by manager_external_id → map to user_id if exists; if not found, leave manager_id null and log warning.
    - Sync actions: create, update, deactivate (account_enabled=false), and "no-op" when no changes detected.
    - Error handling: invalid emails, duplicate emails, missing required fields.

    Keep the doc under ~2 pages and structured with tables and bullet lists so it can be read during implementation without ambiguity.
  </action>

<task type="auto">
  <name>Task 3: Implement sync service and directory API endpoints</name>
  <files>backend/app/services/directory_sync_service.py, backend/app/api/v1/endpoints/directory.py, backend/app/api/v1/router.py</files>
  <action>
    Build a sync service that:
    - Loads DirectoryUser records in batches.
    - Normalizes emails to lowercase and trims whitespace.
    - Matches existing User by email, fallback to UPN.
    - Creates new users when no match exists, defaulting role_id to RoleType.EMPLOYEE.
    - Resolves department: match by name; create if missing.
    - Resolves manager: match by manager_external_id to a DirectoryUser linked to a User; set manager_id if found.
    - Updates existing users: name (prefer display_name else given+surname), email, department_id, manager_id, is_active.
    - Does not overwrite role_id unless a mapping table is introduced later.
    - Collects per-user diffs for preview and logs results in DirectorySyncLog.

    Implement endpoints under /api/v1/directory (admin-only):
    - GET /directory/users (filters: email, department, active)
    - POST /directory/users (create)
    - PATCH /directory/users/{id} (update)
    - DELETE /directory/users/{id} (soft delete => account_enabled=false)
    - POST /directory/sync/preview (returns diffs without writing users)
    - POST /directory/sync/apply (applies sync and returns counts)
    - GET /directory/sync/history (last 10 logs)

    Reuse auth and permission checks from users endpoints (can_manage_users).
    Register router in api/v1/router.py.
  </action>
  <verify>curl /api/v1/directory/users and /api/v1/directory/sync/preview returns 200</verify>
  <done>Sync service updates users as expected and endpoints return correct payloads</done>
</task>
