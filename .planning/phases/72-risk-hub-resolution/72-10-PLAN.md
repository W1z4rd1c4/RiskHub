<task type="auto">
  <name>Task 2: Add `GET /riskhub/public-risk-types` (active types only)</name>
  <files>backend/app/api/v1/endpoints/riskhub.py</files>
  <action>
    Add a new authenticated endpoint that is NOT CRO-only:
    - `GET /riskhub/public-risk-types`

    Requirements:
    - Return only active risk types (`RiskTypeConfig.is_active == True`)
    - Return only the fields needed for display:
      - `code`, `display_name`, `color`, `icon`, `sort_order`
    - Do not return CRO-only admin metadata (created_at, updated_by, is_system flags, etc) unless frontend truly needs it.

    Implementation sketch:
    - Define a lightweight Pydantic schema near the endpoint:
      - `class PublicRiskTypeRead(BaseModel): code: str; display_name: str; color: str; icon: str|None; sort_order: int`
    - Query:
      - `select(RiskTypeConfig).where(RiskTypeConfig.is_active == True).order_by(RiskTypeConfig.sort_order, RiskTypeConfig.display_name)`
  </action>
  <verify>`rg -n \"public-risk-types\" backend/app/api/v1/endpoints/riskhub.py`</verify>
  <done>All authenticated users can list active risk types for UI display.</done>
</task>
