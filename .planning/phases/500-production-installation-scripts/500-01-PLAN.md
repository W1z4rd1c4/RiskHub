<objective>
Define the production configuration contract for split backend/frontend containers with externally managed PostgreSQL.
</objective>

<context>
This Phase 500 install path is opinionated and must remain consistent:
- PostgreSQL is external (no DB container is started by RiskHub scripts).
- Redis is required in production (backend startup fails without it).
- Production auth is SSO-only: `AUTH_MODE=microsoft_sso` when `DEBUG=false`.
- Same-host topology: frontend proxies `/api` to backend via Docker network alias `backend`.
- Bootstrap requirement: SSO JIT provisioning uses a safe default role (not admin). Pre-provision an initial admin/CRO user by email so first SSO login can bind `external_id`.
</context>

<tasks>

<task type="auto">
  <name>Task 1: Create backend environment template for external PostgreSQL deployment</name>
  <files>scripts/prod/config/backend.env.example</files>
  <action>
Create a backend env template that explicitly requires:
- `DATABASE_URL` pointing to external PostgreSQL host (must not be the default placeholder; must not use `db` hostname).
- Production-safe defaults:
  - `DEBUG=false`
  - `MOCK_AUTH_ENABLED=false`
  - `AUTH_MODE=microsoft_sso`
- Required security/runtime settings:
  - `SECRET_KEY` (>= 32 chars)
  - `CORS_ORIGINS` (explicit JSON array allowlist)
  - `REDIS_PASSWORD` (used by Phase 500 Redis container)
  - `REDIS_URL` (required at runtime; may be empty in `backend.env` because scripts compute/inject it)
    - computed value: `redis://:<REDIS_PASSWORD>@redis:6379/0` (must point to Docker alias `redis`)
  - `ENTRA_TENANT_ID`, `ENTRA_CLIENT_ID`
- Recommended defaults for this install path:
  - `DIRECTORY_WEBHOOK_ENABLED=false` (set true only if you are actually using directory webhooks)
  - `AD_EMULATOR_URL=` (blank unless needed)

Do not include local compose-only DB variables as required runtime inputs for this path.
  </action>
  <verify>Template clearly marks required values and disallows placeholder defaults in production usage examples.</verify>
  <done>Backend production env contract is explicit and external-DB-first.</done>
</task>
