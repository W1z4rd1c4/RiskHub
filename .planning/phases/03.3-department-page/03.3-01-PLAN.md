# Phase 3.3-01: Department Detail Pages

**Status:** Ready for Execution  
**Created:** 2025-12-25  
**Estimated Scope:** Medium (8-12 tool calls)

## Objective

Create dedicated department detail pages that provide a comprehensive view of each department's risk and control landscape, including key metrics, risk/control lists, and visual analytics. Transform the existing stub `DepartmentsPage` into a functional list view with drill-down capability to individual department pages.

## Execution Context

**Prerequisites:**
- Phase 3.2 complete (list enhancements with grouped views)
- Existing department model and basic endpoints
- Dashboard aggregation patterns established

**Key Files:**
- Backend: `app/api/v1/endpoints/departments.py`, `app/schemas/department.py`
- Frontend: `pages/DepartmentsPage.tsx`, `pages/DepartmentDetailPage.tsx` (new)
- Components: Reuse `MiniHeatmap`, `SortableTable`, dashboard charts

## Context

### Current State
- **Backend:** Basic `/departments` endpoint returns id, name, code only
- **Frontend:** Stub `DepartmentsPage` with hardcoded mock data (3 departments)
- **Navigation:** Department link exists in sidebar but page is non-functional

### What's Missing
1. Department detail endpoint with aggregated metrics
2. Department detail page component
3. Integration with existing risk/control data
4. Visual analytics for department-specific insights

### Design Decisions
- Reuse existing dashboard metric patterns for consistency
- Leverage `CategoryDrillDown` and `MiniHeatmap` components
- Show both high-level stats and detailed risk/control lists
- Include department-specific risk heatmap and control effectiveness charts

## Tasks

### Backend (3 tasks)

#### 1. Create Department Schemas
**File:** `backend/app/schemas/department.py`
- `DepartmentBase`: name, code, description
- `DepartmentRead`: includes id, timestamps
- `DepartmentSummary`: includes counts (users, risks, controls)
- `DepartmentDetail`: includes full metrics (risk distribution, control stats, recent executions)

#### 2. Enhance Department Endpoints
**File:** `backend/app/api/v1/endpoints/departments.py`
- Update `GET /departments` to return `DepartmentSummary[]` with counts
- Add `GET /departments/{id}` returning `DepartmentDetail` with:
  - Basic info (name, code, description)
  - Risk metrics (total, by status, by level, distribution matrix)
  - Control metrics (total, by status, by form, avg effectiveness)
  - Recent control executions (last 10)
  - User count

#### 3. Add Department-Specific List Endpoints
**File:** `backend/app/api/v1/endpoints/departments.py`
- Add `GET /departments/{id}/risks` - filtered risk list for department
- Add `GET /departments/{id}/controls` - filtered control list for department

### Frontend (4 tasks)

#### 4. Create Department API Service
**File:** `frontend/src/services/departmentApi.ts`
- `getDepartments()` - list with summary stats
- `getDepartment(id)` - detail with full metrics
- `getDepartmentRisks(id, params)` - paginated risks
- `getDepartmentControls(id, params)` - paginated controls

#### 5. Update Departments List Page
**File:** `frontend/src/pages/DepartmentsPage.tsx`
- Replace mock data with live API call
- Show department cards with real metrics (users, risks, controls)
- Add mini heatmap to each card showing risk distribution
- Make cards clickable to navigate to detail page
- Add loading/error states

#### 6. Create Department Detail Page
**File:** `frontend/src/pages/DepartmentDetailPage.tsx` (NEW)
- Header with department name, code, description
- Stats row: Total Risks, Total Controls, Active Users, High-Risk Count
- Risk Distribution Matrix (5x5 heatmap, larger than mini version)
- Control Effectiveness Chart (bar chart by form/frequency)
- Tabbed view:
  - **Risks Tab:** `SortableTable` with department's risks
  - **Controls Tab:** `SortableTable` with department's controls
  - **Recent Activity Tab:** Recent control executions
- Back button to departments list

#### 7. Add Routing and Navigation
**Files:** `frontend/src/App.tsx`, `frontend/src/pages/index.ts`
- Add route: `/departments/:id` → `DepartmentDetailPage`
- Export `DepartmentDetailPage` from pages index
- Ensure sidebar link works correctly
