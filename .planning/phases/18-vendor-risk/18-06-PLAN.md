<objective>
Support PI‑18‑03 contract governance requirements for ICT vendors:
- Track required contract clauses/controls (OS‑16 + DORA addenda references) as structured checklist items.
- Record compliance status and evidence references (links/paths) per clause.
- Make gaps visible and auditable, without building full contract lifecycle management.
</objective>

<context>
@.planning/phases/18-vendor-risk/18-RESEARCH.md
@backend/app/models/control_execution.py
</context>

<tasks>

<task type="auto">
  <name>Task 1: Define contract control templates (standard vs DORA)</name>
  <action>
Create template sets (stable keys) for contract controls:
- `ict_standard`
- `ict_dora_non_significant`
- `ict_dora_significant`

Each control item should have:
- `key`
- `title_key` (for FE i18n)
- `description_key` (optional)
- `applies_when` rules explicitly tied to Phase 18 manual flags:
  - `vendor.vendor_type == "ict"` (or equivalent) gates the entire checklist
  - `vendor.dora_relevant` selects DORA vs non-DORA sets
  - `vendor.is_significant_vendor` selects significant vs non-significant DORA sets

Keep v1 content minimal but representative (audit rights, incident reporting SLAs, data location, subcontractor obligations, exit provisions).
  </action>
  <done>Templates exist and can be applied consistently to vendors based on their classification.</done>
</task>

<task type="auto">
  <name>Task 2: Backend model + endpoints — VendorContractControl</name>
  <action>
Add per-vendor checklist records:
- `vendor_id`
- `control_key`
- `status` (met / partial / missing / n_a)
- `evidence_reference` (string) + optional `notes`
- `last_reviewed_at` + `reviewed_by_user_id`

API:
- `GET /vendors/{vendor_id}/contract-controls` (computed “applies” based on vendor classification)
- `PATCH /vendors/{vendor_id}/contract-controls` (bulk update statuses/notes)
  </action>
  <verify>When vendor classification changes, checklist applicability updates deterministically; previously answered items remain stored but show as `n_a` when not applicable (no silent data loss).</verify>
  <done>Contract control compliance can be tracked and reviewed per vendor with stable keys.</done>
</task>

<task type="auto">
  <name>Task 3: UI — contract controls tab</name>
  <action>
Add a “Contract Controls” tab:
- Group by template/section (standard vs DORA).
- Show status pills and evidence link field.
- Only allow edits for authorized users:
  - outsourcing owner, OR
  - users with `vendor_contracts:write` (the “legal/compliance capability”)
  </action>
  <done>Users can record and review contract clause compliance gaps in a structured, auditable manner.</done>
</task>
