<objective>
Implement PI‑18‑03 due diligence as a workflow:
- Outsourcing owner (PI “sponsor”) performs assessment using a structured questionnaire + evidence references.
- Outputs are reviewable by 2nd line (Risk Manager/Compliance) and can be escalated to a “risk committee recommendation” decision.
- Support DORA‑relevant vendors with a longer/more exhaustive questionnaire scope.
</objective>

</context>

<tasks>

<task type="auto">
  <name>Task 1: Assessment model + lifecycle</name>
  <action>
Add `VendorAssessment` with:
- `vendor_id`
- `status` (draft → submitted → in_review → committee_recommended → approved/rejected)
- `template_key` + `template_version`
- `scope` (standard vs dora) derived from `vendor.dora_relevant`
- `answers_json` (stable keys)
- `submitted_at`, `reviewed_at`, `decision_at`
- `submitted_by_user_id`, `reviewed_by_user_id`, `decided_by_user_id`
- `committee_recommendation` (approve/approve_with_conditions/reject) + optional `conditions_text`

Evidence:
- Start with “evidence_reference” strings (URLs/paths) similar to control execution evidence patterns.
- Keep it simple: a single `evidence_reference` text field per assessment in MVP (no evidence taxonomy, no uploads).
- Freeze evidence on submission/decision (immutable snapshot after submit, consistent with Phase 18 context).
  </action>
  <done>Assessment instances are immutable audit records with clear workflow states and evidence references.</done>
</task>

<task type="auto">
  <name>Task 2: Questionnaire templates (standard vs DORA)</name>
  <action>
Define stable question keys grouped by PI themes:
- Financial stability
- Regulatory/legal (GDPR, DORA, Solvency II)
- Cyber/security + supply chain
- BCP/DR + outage tolerance
- Governance/controls + auditability
- Concentration/replaceability + subcontractors (4th parties)

Template rules:
- `standard` template is required for all vendors.
- `dora` template extends standard with additional DORA-focused sections and explicit subcontractor disclosure.

Implementation approach:
- Reuse the “template key + version + answers_json” strategy used for risk questionnaires, but do not require a vendor login portal in this phase.
  </action>
  <done>Standard and DORA assessment templates are defined as stable key sets, ready for scoring and reporting.</done>
</task>

<task type="auto">
  <name>Task 3: API endpoints + permissions</name>
  <action>
Endpoints:
- `GET /vendors/{vendor_id}/assessments`
- `POST /vendors/{vendor_id}/assessments` (create draft)
- `POST /vendor-assessments/{id}/submit`
- `POST /vendor-assessments/{id}/review` (2nd line)
- `POST /vendor-assessments/{id}/committee-recommend` (risk committee recommendation)
- `POST /vendor-assessments/{id}/decide` (CRO/board-like decision; align with RBAC roles available)

RBAC:
- Outsourcing owner can create/edit drafts and submit.
- Risk Manager/Compliance can review and recommend.
- CRO can decide (or map to a configured committee role).
  </action>
  <verify>Illegal transitions (e.g., edit after submit, decide without review) are blocked with clear 400/403 errors.</verify>
  <done>Due diligence workflow works end-to-end with correct role gating and state transitions.</done>
</task>
