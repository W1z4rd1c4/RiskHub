## Objective

Complete the incomplete Phase 151-13 work: implement KRI value correction UI and overdue badges.

## Context

From ROADMAP.md Phase 151:
> `[ ] 151-13: KRI value correction UI + overdue badges`

This plan addresses:

1. UI for correcting previously submitted KRI values
2. Visual badges indicating overdue KRI submissions
3. Integration with the approval workflow (corrections require CRO approval per 157-02)

## Tasks

<task id="1" title="Create KRIValueCorrectionModal component">
Create `frontend/src/components/kri/KRIValueCorrectionModal.tsx`:

```tsx
interface KRIValueCorrectionModalProps {
  kri: KRI;
  historyEntry: KRIValueHistory;
  isOpen: boolean;
  onClose: () => void;
  onSuccess: () => void;
}
```

Features:

- Show current value and period
- Input for new corrected value
- Mandatory correction reason field
- Warning that correction requires CRO approval
- Submit calls `PATCH /api/v1/kris/{id}/history/{entry_id}/correct`
</task>

<task id="2" title="Add correction button to KRI history table">
In `KRIDetailPage.tsx` or history component:
- Add "Correct" button for each historical value row
- Only visible to:
  - Reporting Owner
  - Risk Owner (fallback)
  - Privileged users
- Opens the KRIValueCorrectionModal
</task>

<task id="3" title="Create KRIOverdueBadge component">
Create `frontend/src/components/kri/KRIOverdueBadge.tsx`:

```tsx
interface KRIOverdueBadgeProps {
  kri: KRI;
  className?: string;
}
```

Logic:

- Calculate if current period is past due date
- Show amber/red badge based on days overdue
- Tooltip with days overdue and last submission date
</task>

<task id="4" title="Integrate overdue badges in KRI list and detail views">
