## Objective

Verify and enforce that KRI history corrections require CRO (privileged) approval as documented in BUSINESS_LOGIC.md §5.3.

## Context

**BUSINESS_LOGIC.md §5.3 states:**
> "KRI History Correction → Risk Owner approval → **CRO approval required**"

This means KRI value corrections (not initial submissions) must go through the full tiered approval:

1. Primary approver (Risk Owner) must approve
2. Then escalate to privileged approval (requires_privileged_approval = true)

<task id="3" title="Enforce CRO approval for corrections">
If not already enforced, modify the correction endpoint to:

```python
# When creating approval for KRI correction
approval = ApprovalRequest(
    resource_type=ApprovalResourceType.KRI,
    resource_id=kri.id,
    action_type=ApprovalActionType.EDIT,
    requested_by_id=current_user.id,
    reason=f"KRI value correction: {reason}",
    primary_approver_id=risk_owner_id,
    requires_privileged_approval=True,  # ALWAYS true for corrections
    status=ApprovalStatus.PENDING,
    pending_changes={"value": {"old": old_value, "new": new_value}},
)
```

</task>
