## Objective

Fix the approval cancellation endpoint to allow privileged users to cancel any pending approval, as documented in BUSINESS_LOGIC.md §5.5.

## Context

**BUSINESS_LOGIC.md §5.5 states:**
> "Request creator OR privileged users can cancel PENDING/PENDING_PRIVILEGED requests"

**Current Implementation** (`approvals.py` line 478-479):

```python
if approval.requested_by_id != current_user.id:
    raise HTTPException(status_code=403, detail="Only the requester can cancel their request")
```

**Gap**: Privileged users currently CANNOT cancel other users' pending requests as documented.

## Tasks

<task id="1" title="Fix cancel endpoint authorization">
Modify the `/approvals/{approval_id}/cancel` endpoint in `backend/app/api/v1/endpoints/approvals.py` to check:

1. If user is the original requester → allow cancel
2. OR if user is privileged (`can_resolve_approvals(current_user)`) → allow cancel
3. Otherwise → deny with 403

Replace:

```python
if approval.requested_by_id != current_user.id:
    raise HTTPException(status_code=403, detail="Only the requester can cancel their request")
```

With:

```python
if approval.requested_by_id != current_user.id and not can_resolve_approvals(current_user):
    raise HTTPException(status_code=403, detail="Only the requester or privileged users can cancel requests")
```

</task>

<task id="2" title="Update activity log description for privileged cancellation">
Update the activity log description to differentiate between:
- Self-cancellation: "Approval request cancelled by requester"
- Privileged cancellation: "Approval request cancelled by {user_name} (privileged)"
</task>

<task id="3" title="Add unit tests for privileged cancellation">
Add tests in `backend/tests/test_approvals.py`:

1. `test_privileged_user_can_cancel_other_users_pending_request` - CRO can cancel dept head's pending request
2. `test_risk_manager_can_cancel_other_users_pending_request` - Risk Manager can cancel employee's request
3. `test_non_privileged_cannot_cancel_other_users_request` - Dept head cannot cancel other dept head's request
4. `test_privileged_user_cannot_cancel_already_resolved` - Cannot cancel APPROVED/REJECTED
</task>

<task id="4" title="Verify and commit">
