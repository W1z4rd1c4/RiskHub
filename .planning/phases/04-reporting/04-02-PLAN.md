# Plan: Audit Trail & Control Execution Logging

## Objective

Build UI for logging control executions and viewing audit trail history. Enable managers and auditors to verify control compliance.

---

## Scope

**In Scope:**
- Control execution logging UI (modal form)
- Execution history table
- Audit trail API endpoints
- Filter executions by date, result, control

**Out of Scope:**
- File upload for evidence (future)
- Automated execution detection (future)

---

## Context

The `ControlExecution` model already exists with fields:
- `control_id`, `executed_by_id`, `executed_at`
- `result` (passed/failed/warning/not_applicable)
- `findings`, `evidence_reference`, `notes`
- `next_scheduled`

---

## Tasks

### Task 1: Create Execution Endpoints
Add CRUD endpoints for control executions.

**Files:**
- `backend/app/api/v1/endpoints/executions.py` [NEW]
- `backend/app/api/v1/router.py` [MODIFY]
- `backend/app/schemas/execution.py` [NEW]

**Endpoints:**
| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/v1/executions` | Log a new execution |
| GET | `/api/v1/executions` | List executions (paginated) |
| GET | `/api/v1/executions/{id}` | Get execution details |
| GET | `/api/v1/controls/{id}/executions` | Get executions for a control |

**Query Parameters for List:**
- `control_id` (optional)
- `from_date`, `to_date` (optional)
- `result` (optional)
- `page`, `page_size`
