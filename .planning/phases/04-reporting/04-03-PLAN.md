<task type="auto">
  <name>Task 2: Implement as-of snapshot reconstruction with RBAC-scoped source sets</name>
  <files>backend/app/services/export_snapshot_service.py, backend/app/api/v1/endpoints/reports.py</files>
  <action>
Create export snapshot service used by all 4 entity exporters.

Rules:
1. Build source dataset with the same access scope as list endpoints (department/owner/reporting-owner rules); never bypass existing scoping logic.
2. Always start from include_archived=true source rows to allow replay of historical status transitions.
3. Reconstruct point-in-time state at end-of-day(as_of_date) using reverse replay of ActivityLog changes for entity_type + entity_id where log.created_at > as_of_cutoff:
   - UPDATE/STATUS_CHANGE/ARCHIVE/DELETE: restore changed fields to changes[field].old
   - CREATE after cutoff: drop row from snapshot
4. For KRIs, compute current_value as-of cutoff from latest history entry with period_end <= as_of_date when available; fallback to reconstructed row value when no history exists.
5. After replay, apply user-selected status/search filters and return only visible rows.

Do not attempt to reconstruct users/departments; export only from existing scoped entities.
  </action>
  <verify>As-of export for an entity changed after the selected date returns pre-change status/fields in fixture-based tests.</verify>
  <done>As-of date produces deterministic snapshots without widening access scope.</done>
</task>
