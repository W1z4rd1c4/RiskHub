# Phase 99-01: Risks Migration

## Objective
Migrate all risks from `Registr_Rizik_2022.xlsx` (Rizika sheet) to the database, respecting existing departments and users.

## Execution Context
- Source: `Registr_Rizik_2022.xlsx`, sheet `Rizika`, rows 9+
- Existing script: `backend/scripts/migrate_risks.py` (needs verification)
- Models: `Risk`, `Department`, `User`

## Column Mapping (Excel → Database)

| Excel Column | DB Field | Notes |
|---|---|---|
| B: Hlavní proces | process | Also maps to department |
| C: Podproces | subprocess | |

| E: Solvency II | (unused) | |

| I: Pravděpodobnost (gross) | gross_probability | 1-5 |
| L: Dopad (net) | net_impact | 1-5 |
| M: Pravděpodobnost (net) | net_probability | 1-5 |

## Tasks

- [ ] 1. Verify existing seeded departments or create new ones from `Hlavní proces`
- [ ] 2. Run existing `migrate_risks.py` script with updated Excel path
- [ ] 3. Verify risks are in database with correct department associations
- [ ] 4. Assign risk owners to appropriate users based on department

## Verification
```bash
# Check risks count
docker exec riskhub-db psql -U riskhub -c "SELECT COUNT(*) FROM risks;"
# Verify department linkage
docker exec riskhub-db psql -U riskhub -c "SELECT d.name, COUNT(r.id) FROM risks r JOIN departments d ON r.department_id = d.id GROUP BY d.name;"
```

## Success Criteria
- All ~80 risks from Excel imported
- Each risk linked to correct department
- Gross/net probability and impact scores preserved
