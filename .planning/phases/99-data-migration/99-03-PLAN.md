# Phase 99-03: KRIs Migration and Risk-KRI Linkage

## Objective
Migrate Key Risk Indicators from `Register rizik - limity - Q3.xlsx` and link them to existing risks.

## Execution Context
- Source: `Register rizik - limity - Q3.xlsx`
- Sheets: `Provozní riziko` (44 rows), `Neživotní upisovací riziko`, `Zdravotní upisovací riziko`, `Tržní riziko`, `Riziko selhání protistrany`
- Models: `KeyRiskIndicator`, `Risk`

## Column Mapping (Excel → Database)

| Excel Column (CZ) | DB Field | Notes |
|---|---|---|
| ID | (reference only) | |
| Hlavní proces | (lookup risk) | Match to Risk.process |
| Klíčová rizika - popis | (lookup risk) | Match to Risk.description |
| Metrika | metric_name | |
| Hodnota | current_value | |
| Dolní limit | lower_limit | |
| Horní limit | upper_limit | |
| (infer from metric) | unit | Default '%' or 'count' |

## Risk Matching Strategy
1. Match by `Klíčová rizika - popis` against `Risk.description` (fuzzy match)
2. Fall back to `Hlavní proces` = `Risk.process` + manual linkage
3. Create orphaned KRIs with null risk_id if no match (flag for review)

## Tasks

- [ ] 1. Create `migrate_kris.py` script for KRI import
- [ ] 2. Iterate all risk-related sheets (Provozní, Neživotní, etc.)
- [ ] 3. Match KRIs to imported risks by description/process
- [ ] 4. Import KRIs with correct risk_id foreign keys
- [ ] 5. Report unmatched KRIs for manual review
