<task type="auto">
  <name>Task 3: Create AD Emulator API endpoints</name>
  <files>AD Emulator/backend/app/api/__init__.py, AD Emulator/backend/app/api/router.py, AD Emulator/backend/app/api/endpoints/__init__.py, AD Emulator/backend/app/api/endpoints/users.py, AD Emulator/backend/app/api/endpoints/health.py, AD Emulator/backend/app/schemas/__init__.py, AD Emulator/backend/app/schemas/directory_user.py</files>
  <action>
    Create REST API for the AD Emulator that RiskHub will consume:
    
    1. `app/schemas/directory_user.py` - Pydantic schemas:
       - DirectoryUserCreate (for admin UI)
       - DirectoryUserUpdate (for admin UI)
       - DirectoryUserRead (public response)
       - DirectoryUserListResponse (paginated list)
    
    2. `app/api/endpoints/users.py` - Directory user management:
       - GET /users - List all active directory users (RiskHub calls this for sync)
       - GET /users/{external_id} - Get single user by external_id
       - POST /users - Create directory user (admin)
       - PATCH /users/{id} - Update directory user (admin)
       - DELETE /users/{id} - Soft-delete (set account_enabled=false)
       
       These endpoints simulate what Entra ID / Microsoft Graph would expose.
       No auth required for now (internal/dev tool), but add basic API key later.
    
    3. `app/api/endpoints/health.py` - Health check endpoint
    
    4. `app/api/router.py` - Combine all routers
    
    5. Update `app/main.py` to include the router
    
    The key difference from RiskHub's version:
    - No "sync" endpoints - sync is triggered FROM RiskHub, not from AD Emulator
    - AD Emulator just serves directory data for RiskHub to consume
  </action>
