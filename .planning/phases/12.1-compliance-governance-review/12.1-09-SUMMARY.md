# Phase 12.1 Plan 09: Risk Committee Metrics Remediation Summary

**Quarter boundaries corrected, historical snapshot system implemented, frontend hardened for truthful quarter-over-quarter comparisons.**

## Accomplishments

### Quarter Boundary Fixes
- Changed all datetime usage to UTC-aware (`datetime.now(timezone.utc)`)
- Implemented end-exclusive semantics (`[start, end)`) for all period-based queries
- Fixed `last_quarter_end` calculation (was `start - 1 day`, now `current_quarter_start`)

### Historical Snapshot System (User Decision: Option C)
- Created `QuarterlyMetricSnapshot` model with JSON metrics storage
- Added Alembic migration for `quarterly_metric_snapshots` table
- Implemented `snapshot_service.py` for capturing/retrieving snapshots
- Separated metrics into period-based (queryable by time range) and snapshot-based (point-in-time state)

### Dashboard Endpoint Refactoring
- Refactored `/dashboard/quarterly-comparison` to:
  - Query period metrics for both quarters from live data
  - Load snapshot metrics from stored snapshots for historical quarters
  - Fall back gracefully when no historical snapshot exists
- Added `snapshot_info` to response with metadata about data sources
- Replaced string literals with `ApprovalStatus` enum values

### Admin Snapshot Endpoints
- `POST /admin/snapshots/capture` - Manual snapshot capture
- `GET /admin/snapshots` - List all stored snapshots
- `GET /admin/snapshots/{quarter}` - Retrieve specific snapshot

### Query Optimization
- Added index to `ControlExecution.executed_at` for efficient range queries

### Frontend Widget Hardening
- Added proper TypeScript types for API response including `snapshot_info`
- Defensive rendering for missing/unknown metrics (`?? null`, `?? 0`)
- Warning banner when historical snapshot unavailable
- Visual indicators (amber border, help icon) for uncertain comparisons

### Quarterly Report Package Clarification
- Updated Phase 12 context to document implementation:
  - Browser print/PDF via `window.print()`
  - No server-side PDF generation required
  - Admin snapshot capture for accurate historical comparisons
