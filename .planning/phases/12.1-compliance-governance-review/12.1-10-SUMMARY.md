# Phase 12.1 Plan 10: SIEM & Logging Remediation Summary

**Fixed admin log endpoints, middleware attribution, rotation config, and failed-login auditing for operational SIEM reliability.**

## Accomplishments

### Admin Log Endpoints (Tasks 1-2)
- Added `get_log_directory()` helper as single source of truth for log paths
- Implemented `tail_log_file()` for efficient reverse reading (no full-file scans)
- Fixed `TechnicalLogEntry` schema to use `description` (matching ActivityLog model)
- Fixed session revoke endpoint to use correct `log_activity()` signature

### Middleware User Attribution (Task 3)
- Fixed JWT extraction to use `user_id` claim instead of parsing `sub` (email) as int
- Added signature verification to prevent spoofed attribution from invalid tokens
- Invalid/expired tokens now return `None` (no user attribution) instead of crashes

### Log Rotation Config (Task 4)
- Added startup lifespan handler to apply rotation settings from Risk Hub config DB
- Settings now apply on backend restart (documented behavior)
- `configure_logging()` accepts optional `rotation_size_mb` and `retention_count`

### Failed-Login Auditing (Task 5)
- Changed failed logins from `LOGIN` to `FAILED_LOGIN` action
- Unified description ("Failed login attempt: invalid credentials") to avoid leaking user existence
- Removed user ID and actor attribution from failed login logs

### Verification Tooling (Task 6)
- Upgraded `verify_audit_logs.py` with:
  - Logger validation (audit logs must have `logger == "audit"` or `audit.*`)
  - Log separation enforcement (check app log for audit leakage)
  - Structured `VerificationResult` with errors/warnings
  - Improved secret patterns (scoped to avoid false positives)

### Documentation (Task 7)
- Updated `ADMIN_SIEM_GUIDE.md` with correct field documentation
- Added rotation config behavior section (when changes apply)
- Added verification script usage instructions
- Fixed ROADMAP.md and STATE.md to reflect Phase 12.1 completion
