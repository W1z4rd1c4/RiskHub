  <files>backend/app/middleware/logging_context.py, backend/tests/test_siem_logging.py</files>
  <action>
Fix request context attribution:
- Tokens encode `user_id` as a dedicated claim and `sub` as email; middleware currently parses `sub` as an int.
- Update middleware to extract `user_id` from the correct claim.

Security hardening:

- Recommended: verify JWT signature (and expiration) before trusting `user_id` for logging context, or only set `user_id` after auth.

Update/extend `backend/tests/test_siem_logging.py` to assert the expected context fields exist and that user attribution behaves as intended for valid vs invalid tokens.
  </action>
