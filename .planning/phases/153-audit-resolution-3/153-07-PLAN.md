  <name>Task 3: Create NEW migration to add server_default (DO NOT modify existing migration)</name>
  <files>backend/alembic/versions/</files>
  <action>
  **⚠️ CRITICAL: DO NOT modify the existing migration `597c3ba51f80_add_tiered_approval_fields.py`!**
  
  Modifying already-applied migrations causes:
  - Inconsistency between environments (dev/staging/prod)
  - Alembic won't re-run the migration (tracked by revision ID)
  - Confusion about what was actually deployed
  
  **Instead, create a NEW migration to add the server_default:**
  
  ```python
  """add_server_default_to_requires_privileged_approval
  
  Adds server_default to requires_privileged_approval column for safety
  when adding rows to approval_requests table.
  """
  from alembic import op
  import sqlalchemy as sa
  
  revision = 'fix_tiered_default_001'  # Generate unique ID
  down_revision = 'LATEST_MIGRATION'
  
  def upgrade():
      # Add server_default to existing column
      op.alter_column('approval_requests', 'requires_privileged_approval',
                      server_default='false',
                      existing_type=sa.Boolean(),
                      existing_nullable=False)
  
  def downgrade():
      # Remove server_default
      op.alter_column('approval_requests', 'requires_privileged_approval',
                      server_default=None,
                      existing_type=sa.Boolean(),
                      existing_nullable=False)
  ```
  
  This safely adds the default to existing databases without modifying history.
  </action>
