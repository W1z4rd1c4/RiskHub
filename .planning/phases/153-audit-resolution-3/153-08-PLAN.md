<task type="auto">
  <name>Task 1: Add cross-department owner check to update_risk</name>
  <files>backend/app/api/v1/endpoints/risks.py</files>
  <action>
  In the `update_risk` endpoint (starting around line 378), add cross-department ownership checks BEFORE the department access check, matching the pattern in `get_risk`.
  
  **Current code structure (lines 400-413):**
  ```python
  if not risk:
      raise HTTPException(status_code=404, detail="Risk not found")
  
  # Verify department access
  check_department_access(risk.department_id, current_user)  # ❌ Fails for cross-dept owners
  
  # Check permission: either risks:write or is risk owner
  has_write = check_permission(current_user, "risks", "write")
  is_owner = risk.owner_id == current_user.id
  
  if not has_write and not is_owner:
      raise HTTPException(...)
  ```
  
  **Change to (insert owner check before department check):**
  ```python
  if not risk:
      raise HTTPException(status_code=404, detail="Risk not found")
  
  # Check permission: either risks:write or is risk owner
  has_write = check_permission(current_user, "risks", "write")
  is_owner = risk.owner_id == current_user.id
  
  # Risk owners can edit their risk regardless of department (cross-department access)
  if not is_owner:
      # Verify department access only for non-owners
      check_department_access(risk.department_id, current_user)
  
  if not has_write and not is_owner:
      raise HTTPException(
          status_code=status.HTTP_403_FORBIDDEN,
          detail="Permission denied: risks:write or risk owner required"
      )
  ```
  
  **Why this matters:** Per BUSINESS_LOGIC.md §2.1 and §7.1, Risk Owners can edit risks they own regardless of department. The current implementation blocks cross-department owners because `check_department_access` runs before the owner check.
  </action>
