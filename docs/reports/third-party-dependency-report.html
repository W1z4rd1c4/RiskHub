replace (passlib)
The one real problem: passlib. It is unmaintained (last release Oct 2020) and is the sole reason bcrypt must be pinned to 4.1.3 — passlib crashes on bcrypt ≥ 4.1. Only app/core/security.py uses it, so the migration surface is one file. Best replacement: pwdlib — actively maintained, built specifically as passlib's successor, verifies existing bcrypt hashes unchanged (no user re-hash needed), and unlocks Argon2id.
Watch: psycopg2-binary. Only used by the dbtasks image for Alembic migrations and seeding — not the app runtime — so it's low urgency. When convenient, move to psycopg (v3), the maintained successor; Alembic supports it out of the box.

Bottom Line

This is a modern, conservative, well-pinned stack with no exotic dependencies. Backend (FastAPI + SQLAlchemy 2 + Postgres + Redis) and frontend (React 19 + Vite + Tailwind + Radix + TanStack Query) are each the current mainstream best practice, and the supply-chain hygiene (digest-pinned images, explicit Starlette pin, non-root Redis image, Dependabot) is above average. Two actions:

  1. Replace passlib with pwdlib in app/core/security.py — removes an unmaintained dependency and the bcrypt 4.1.3 pin. Existing bcrypt hashes keep verifying.
  2. Eventually swap psycopg2-binary for psycopg (v3) in the dbtasks tooling image — low urgency, tooling-only.
