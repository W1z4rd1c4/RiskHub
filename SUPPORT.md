# Support

RiskHub uses GitHub for public support routing.

## Questions And Self-Hosting Help

Use GitHub Discussions for setup questions, Docker demo help, local evaluation notes, and workflow questions:

- https://github.com/W1z4rd1c4/RiskHub/discussions

Include the command you ran, the deployment mode (`demo`, `dev`, or `production`), and any relevant error text.

## Bugs

Use the bug report template for reproducible defects:

- https://github.com/W1z4rd1c4/RiskHub/issues/new?template=bug_report.md

Include steps to reproduce, expected behavior, actual behavior, environment, and logs or screenshots when useful.

## Feature Requests

Use the feature request template for product or operator workflow improvements:

- https://github.com/W1z4rd1c4/RiskHub/issues/new?template=feature_request.md

Describe the user/operator problem first, then the behavior you want.

## Security Vulnerabilities

Do not open public issues for exploitable vulnerabilities. Use GitHub private vulnerability reporting:

- https://github.com/W1z4rd1c4/RiskHub/security/advisories/new

If private reporting is unavailable, open a minimal public issue requesting a secure contact path and do not include exploit details. See [SECURITY.md](./SECURITY.md).
